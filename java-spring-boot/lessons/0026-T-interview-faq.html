Spring Boot 面試八股文 + Spring vs Spring Boot

後端面試中有一類問題不考程式碼，而是考你對框架「為什麼這樣設計」的理解。這堂課整理最高頻的 Spring Boot 八股題，每題附面試官想聽的答案和你自己在課程中對應的經驗。

🎯 這堂課的定位 面試中回答「為什麼」比回答「是什麼」重要十倍。每題都附一個你在整合專案中的具體例子——讓面試官覺得你不是在背誦，是真的用過。

Part A：Spring Boot vs Spring Framework

Q: Spring Boot 和 Spring Framework 有什麼不同？

面試官想聽：不是單純列舉功能差異，而是說出「Spring Boot 解決了什麼痛點」。

標準回答：Spring Framework 是 Java 後端的核心 DI/AOP 框架，但使用它需要大量手動設定——XML 配置、Servlet 容器設定、依賴版本管理。Spring Boot 是建立在 Spring Framework 之上的自動化層，用三個核心機制解決了這些痛點：自動配置（Auto-configuration，根據 classpath 自動載入設定）、起步依賴（Starter，一組預先測試過相容版本的依賴包）、內嵌伺服器（Embedded Tomcat，一行 run() 就能啟動）。Spring Boot 不是取代 Spring，而是讓 Spring 更好用。

Spring FrameworkSpring Boot
設定方式XML / Java Config（手寫大量 @Bean）Auto-configuration（零設定起手）
依賴管理手動指定每個 library 版本Starter 一鍵引入，版本統一管理
伺服器需要外部 Tomcat，打成 WAR內嵌 Tomcat/Jetty，直接 java -jar
啟動方式部署到 Servlet 容器SpringApplication.run()
監控需手動整合Actuator 內建 /health /metrics
入門時間數天（讀文件、寫設定）數分鐘（start.spring.io）
💡 加分說法 「我之前用 Spring Initializr 建立專案，選 Spring Web + JPA 兩個 starter，一行設定都沒寫就能啟動 REST API——這就是 Spring Boot 的價值。但我也理解底層仍然是 Spring 的 IoC 容器和 AOP 在運作（T05/T06 學的），所以除錯時知道要去看 Bean 註冊和代理機制。」

Q2: @SpringBootApplication 做什麼？

// 你每天寫的：
@SpringBootApplication
public class BlogApiApplication {
    public static void main(String[] args) {
        SpringApplication.run(BlogApiApplication.class, args);
    }
}

// 實際上等於三個 annotation 的組合：
@SpringBootConfiguration   // = @Configuration：這個 class 可以定義 @Bean
@EnableAutoConfiguration   // 自動配置的核心：根據 classpath 猜你需要什麼
@ComponentScan             // 掃描這個 package 以下的所有 @Component/@Service/@Controller
public @interface SpringBootApplication { }
Annotation作用如果沒有會怎樣
@SpringBootConfiguration標記為設定類別不能在此定義 @Bean
@EnableAutoConfiguration自動設定 JPA、Web、Security...必須手動寫所有設定（回到 Spring Framework 時代）
@ComponentScan掃描 Bean你的 @Service、@Controller 不會被註冊

Q3: Spring Boot 的自動配置（Auto-configuration）是怎麼運作的？

標準回答：Spring Boot 在 classpath 中尋找 spring.factories 檔案（Spring Boot 2.x）或 META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports（Spring Boot 3.x），裡面列出了所有自動配置類別。每個自動配置類別上都有 @ConditionalOnClass、@ConditionalOnMissingBean 等條件註解——只在你引入了對應依賴、且你沒有自己定義 Bean 時才會生效。

// 例如 DataSourceAutoConfiguration 的簡化邏輯：
@AutoConfiguration
@ConditionalOnClass({ DataSource.class, EmbeddedDatabaseType.class })
// ↑ 只有 classpath 中有 DataSource 這個 class 時才生效（即引入了 JPA starter）
public class DataSourceAutoConfiguration {

    @Bean
    @ConditionalOnMissingBean
    // ↑ 只有你沒有自己定義 DataSource Bean 時才建立預設的
    public DataSource dataSource() {
        // 根據 application.properties 的 spring.datasource.* 建立 DataSource
    }
}
🔍 你在專案中的經歷 P07 中你引入 spring-boot-starter-data-jpa 後，一行資料庫連線設定都沒寫，Spring Boot 就自動建立了 H2 DataSource、EntityManagerFactory、TransactionManager——這就是自動配置。然後你在 application.properties 中加了 spring.datasource.url=jdbc:h2:mem:taskdb，自動配置就讀取你的設定並套用。

Q4: Spring Boot Starter 是什麼？常見的有哪些？

標準回答：Starter 是一組預先測試過相容版本的依賴集合。例如 spring-boot-starter-web 包含了 Spring MVC、Jackson、內嵌 Tomcat——你不需要自己去找相容的版本號。

Starter包含什麼你在哪堂課用的

Part B：Spring Boot 核心八股題

Q5: application.properties vs application.yml？

# application.properties（你課程中用的）
spring.datasource.url=jdbc:h2:mem:testdb
spring.jpa.show-sql=true
# application.yml（結構化寫法，複雜設定更易讀）
spring:
  datasource:
    url: jdbc:h2:mem:testdb
  jpa:
    show-sql: true

標準回答：功能完全相同，都是 Spring Boot 的外部設定檔。properties 適合簡單的扁平設定，YAML 適合巢狀結構（如多資料來源、CORS 規則）。Spring Boot 啟動時自動載入兩者，如果同時存在，properties 優先。面試中說「團隊統一選一種，大部分用 YAML 因為支援多行字串和巢狀結構」即可。

Q6: Spring Boot 如何讀取外部設定？優先順序是什麼？

標準回答：Spring Boot 支援 17 種設定來源，從高到低的優先順序：

  1. 命令列參數（--server.port=9090）——最高優先
  2. 作業系統環境變數（SPRING_DATASOURCE_URL）
  3. application.properties（在 jar 外）
  4. application.properties（在 jar 內）——最低優先

實務上最重要的三種讀取方式：

// 1. @Value（單一值）
@Value("${spring.datasource.url}")
private String dbUrl;

// 2. @ConfigurationProperties（整組設定，型別安全）
@ConfigurationProperties(prefix = "app.blog")
public record BlogConfig(String title, int pageSize) { }

// 3. Environment（程式化讀取）
@Autowired Environment env;
String port = env.getProperty("server.port");

Q7: Spring Boot Actuator 是什麼？

標準回答：Actuator 是 Spring Boot 內建的生產環境監控模組，提供開箱即用的端點來監控應用健康狀態、metrics、環境變數、日誌等級等。

// build.gradle
implementation 'org.springframework.boot:spring-boot-starter-actuator'

// 常用端點：
// GET /actuator/health      → {"status":"UP"}（負載均衡器用這個判斷服務是否活著）
// GET /actuator/metrics     → JVM 記憶體、CPU、HTTP 請求統計
// GET /actuator/env         → 所有環境變數和設定
// GET /actuator/beans       → 列出 IoC 容器中的所有 Bean（你在 P04 做過！）
// GET /actuator/mappings    → 列出所有註冊的 URL 映射

Q8: 內嵌 Tomcat vs 外部 Servlet 容器？

標準回答：Spring Boot 預設使用內嵌 Tomcat，直接 java -jar 就能啟動，不需要安裝設定獨立的 Tomcat。優點：部署簡單（一個 jar 搞定）、開發環境一致、適合容器化（Docker）。如果需要改用 Jetty 或 Undertow，排除 Tomcat 依賴、引入對應 starter 即可。

// 改用 Jetty（build.gradle）
configurations.all {
    exclude group: 'org.springframework.boot', module: 'spring-boot-starter-tomcat'
}
implementation 'org.springframework.boot:spring-boot-starter-jetty'

Part C：快速搶答題

面試題30 秒答案
Spring Boot 如何排除特定的自動配置？ @SpringBootApplication(exclude = DataSourceAutoConfiguration.class) 或在 properties 中 spring.autoconfigure.exclude=...
如何更換內嵌伺服器的 port？ server.port=9090（properties）或 --server.port=9090（命令列）
什麼是 Spring Initializr？ start.spring.io——線上產生 Spring Boot 專案骨架，選依賴、產生 ZIP
Spring Boot 支援哪些日誌框架？ 預設 Logback，也支援 Log4j2、Java Util Logging。用 starter 自動排除衝突
如何定義多個 Bean 的初始化順序？ @DependsOn("otherBean") 或讓建構子依賴自然排序
Spring Boot 的熱重載（hot reload）？ spring-boot-devtools：改程式碼後自動重啟（非熱部署，是快速重啟）
為什麼 main 方法在 Spring Boot 中不會結束？ 因為內嵌 Tomcat 的非守護執行緒（non-daemon thread）持續執行，JVM 不會退出

Spring Boot vs Spring：面試一句話總結

「Spring Framework 是引擎（DI + AOP 核心），Spring Boot 是整車（自動配置 + Starter + 內嵌伺服器）。沒有 Spring Framework，Spring Boot 無法運作；沒有 Spring Boot，Spring Framework 需要大量手動設定才能上路。我在課程中用 Spring Initializr 建立專案、一行設定都沒寫就啟動了 REST API——這就是 Spring Boot 的價值。」

自我檢驗 🧠

Q1: 面試官問「Spring Boot 和 Spring 有什麼不同」，你怎麼用一句話切入？

答案

Spring Boot 是 Spring Framework 的自動化層，用自動配置、Starter 依賴、內嵌伺服器三個機制，把 Spring 從「需要大量手動設定」變成「幾乎零設定」。它沒有取代 Spring，而是讓 Spring 的 IoC/AOP 開箱即用。

Q2: @SpringBootApplication 背後是哪三個 annotation 的組合？

答案

@SpringBootConfiguration（標記設定類別）+ @EnableAutoConfiguration（觸發自動配置）+ @ComponentScan（掃描 Bean）。三者缺一不可：缺自動配置你必須手寫所有 @Bean，缺 ComponentScan 你的 @Service 不會被發現。

Q3: 你在自己的 blog-api 專案中，哪裡直接體會到了 Spring Boot 自動配置的價值？

答案

P07 引入 spring-boot-starter-data-jpa 後，只定義了一個 JpaRepository interface，Spring Boot 自動建立了 DataSource、EntityManagerFactory、TransactionManager——我一行 SQL 連線設定都沒寫。P10 引入 Spring Security starter 後，密碼自動被 BCrypt 加密——我只需要注入 PasswordEncoder。
