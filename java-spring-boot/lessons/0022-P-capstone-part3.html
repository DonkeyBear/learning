整合專案 Part 3：Comment + N+1 查詢 + Swagger

最後一堂整合實作課。加入 Comment 巢狀關聯、解決 JPA 經典的 N+1 查詢問題、並整合 Swagger 自動產生 API 文件——讓你的專案在面試中看起來更專業。

🎯 Part 3 目標 Comment Entity（自參照巢狀回覆）→ 解決 N+1 問題（@EntityGraph）→ Swagger/SpringDoc 整合 → 專案完工。

Step 1：Comment Entity（自參照巢狀結構）

comment/Comment.java——Comment 可以回覆另一個 Comment：

package com.blogapi.comment;

import com.blogapi.post.Post;
import com.blogapi.user.User;
import jakarta.persistence.*;
import java.time.LocalDateTime;
import java.util.*;

@Entity
@Table(name = "comments")
public class Comment {

    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, columnDefinition = "TEXT")
    private String content;

    @Column(name = "created_at", nullable = false)
    private LocalDateTime createdAt;

    // 屬於哪篇文章
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "post_id", nullable = false)
    private Post post;

    // 誰寫的
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "author_id", nullable = false)
    private User author;

    // 自參照：回覆哪個 comment（null = 頂層留言）
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "parent_id")
    private Comment parent;

    // 這則留言的所有回覆
    @OneToMany(mappedBy = "parent", cascade = CascadeType.ALL)
    @OrderBy("createdAt ASC")
    private List<Comment> replies = new ArrayList<>();

    protected Comment() { }

    public Comment(String content, Post post, User author, Comment parent) {
        this.content = content;
        this.post = post;
        this.author = author;
        this.parent = parent;
        this.createdAt = LocalDateTime.now();
    }

    public Long getId() { return id; }
    public String getContent() { return content; }
    public LocalDateTime getCreatedAt() { return createdAt; }
    public Post getPost() { return post; }
    public User getAuthor() { return author; }
    public Comment getParent() { return parent; }
    public List<Comment> getReplies() { return replies; }
}

Step 2：Comment Repository + DTO

comment/CommentRepository.java：

package com.blogapi.comment;

import org.springframework.data.jpa.repository.*;
import java.util.List;

public interface CommentRepository extends JpaRepository<Comment, Long> {

    // 查文章的頂層留言（parent = null），並預載 author
    @EntityGraph(attributePaths = {"author", "replies", "replies.author"})
    List<Comment> findByPostIdAndParentIsNullOrderByCreatedAtDesc(Long postId);
}
🔥 @EntityGraph 解決 N+1 問題 如果沒有 @EntityGraph，JPA 會：先查 Comment（1 條 SQL），然後對每個 Comment 的 author 各發一條 SQL（N 條），對每個 Comment 的 replies 再發 N 條——這就是 N+1 問題。

@EntityGraph 告訴 JPA：「這條查詢中順便 JOIN 這些關聯欄位」，用一條 SQL 全部載入。這是面試中最常見的 JPA 效能問題解法。

comment/dto/CommentResponse.java（遞迴巢狀）：

package com.blogapi.comment.dto;

import com.blogapi.comment.Comment;
import java.time.LocalDateTime;
import java.util.List;

public record CommentResponse(
    Long id, String content, LocalDateTime createdAt,
    String authorName, String authorUsername,
    Long parentId,
    List<CommentResponse> replies
) {
    public static CommentResponse from(Comment comment) {
        return new CommentResponse(
            comment.getId(), comment.getContent(), comment.getCreatedAt(),
            comment.getAuthor().getDisplayName(),
            comment.getAuthor().getUsername(),
            comment.getParent() != null ? comment.getParent().getId() : null,
            comment.getReplies().stream()
                .map(CommentResponse::from)   // ← 遞迴轉換
                .toList()
        );
    }
}

Step 3：Comment Service + Controller

comment/CommentService.java：

package com.blogapi.comment;

import com.blogapi.comment.dto.*;
import com.blogapi.post.PostRepository;
import com.blogapi.user.UserRepository;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.util.List;

@Service
public class CommentService {
    private final CommentRepository commentRepo;
    private final PostRepository postRepo;
    private final UserRepository userRepo;

    public CommentService(CommentRepository commentRepo,
                          PostRepository postRepo, UserRepository userRepo) {
        this.commentRepo = commentRepo;
        this.postRepo = postRepo;
        this.userRepo = userRepo;
    }

    @Transactional
    public CommentResponse create(Long postId, CreateCommentRequest req) {
        var post = postRepo.findById(postId)
            .orElseThrow(() -> new IllegalArgumentException("文章不存在"));
        var author = userRepo.findById(req.authorId())
            .orElseThrow(() -> new IllegalArgumentException("使用者不存在"));

        Comment parent = null;
        if (req.parentId() != null) {
            parent = commentRepo.findById(req.parentId())
                .orElseThrow(() -> new IllegalArgumentException("父留言不存在"));
        }

        return CommentResponse.from(
            commentRepo.save(new Comment(req.content(), post, author, parent)));
    }

    public List<CommentResponse> getByPost(Long postId) {
        return commentRepo.findByPostIdAndParentIsNullOrderByCreatedAtDesc(postId)
            .stream().map(CommentResponse::from).toList();
    }
}

comment/CommentController.java：

package com.blogapi.comment;

import com.blogapi.comment.dto.*;
import com.blogapi.common.ApiResponse;
import jakarta.validation.Valid;
import org.springframework.http.HttpStatus;
import org.springframework.web.bind.annotation.*;
import java.util.List;

@RestController
@RequestMapping("/api/posts/{postId}/comments")
public class CommentController {
    private final CommentService commentService;

    public CommentController(CommentService commentService) {
        this.commentService = commentService;
    }

    @GetMapping
    public ApiResponse<List<CommentResponse>> list(@PathVariable Long postId) {
        return ApiResponse.ok(commentService.getByPost(postId));
    }

    @PostMapping
    @ResponseStatus(HttpStatus.CREATED)
    public ApiResponse<CommentResponse> create(
        @PathVariable Long postId,
        @Valid @RequestBody CreateCommentRequest req
    ) {
        return ApiResponse.ok(commentService.create(postId, req));
    }
}

Step 5：效能驗證 — 觀察 N+1 的消失

在 application.properties 中確認 SQL logging 已開啟：

spring.jpa.show-sql=true

訪問 GET /api/posts/1/comments，觀察 Console 的 SQL 數量：

  • 有 @EntityGraph：1 條 SQL（JOIN author + replies）
  • 若移除 @EntityGraph：1 + N 條 SQL（這就是 N+1）

你可以試著把 @EntityGraph 註解掉，重啟，再訪問同一條 API——親眼看到 SQL 暴增。
