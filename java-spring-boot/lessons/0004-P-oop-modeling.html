OOP 建模練習：設計小型後端分層模型

這堂實作課會把 T02 學到的 class、abstract class、interface、record 全部串在一起，模擬一個 Spring Boot 後端常見的分層模型。還沒有用到 Spring——純 Java 練習，但架構和命名會刻意模仿 Spring Boot 專案的真實模樣。

🎯 練習目標 建立一個「圖書館管理系統」的模型層，包含：一個 record（DTO）、一個 abstract class（基底 Entity）、一個 interface（借閱合約）、兩個 concrete class（實體書/電子書）、和一個 Service class（業務邏輯）。

專案結構預覽

完成後的 src/ 目錄會長這樣：

src/
├── Main.java                  ← 進入點，測試所有類別
├── model/
│   ├── BookDto.java           ← record（DTO）
│   ├── BaseEntity.java        ← abstract class
│   └── LibraryItem.java       ← interface
├── entity/
│   ├── PhysicalBook.java      ← extends BaseEntity implements LibraryItem
│   └── EBook.java             ← extends BaseEntity implements LibraryItem
└── service/
    └── LibraryService.java    ← 管理借閱邏輯

這其實就是 Spring Boot 的標準目錄結構——model/、entity/、service/，只是現在還沒有 Spring 的 annotation。

💡 建立 package 的方法 在 IntelliJ 中，對 src 目錄按右鍵 → New → Package → 輸入 model。然後在 package 上按右鍵 → New → Java Class 來建立新檔案。Package 名稱全小寫，和目錄結構對應。

Step 1：建立 BookDto（Record）

這是對外傳輸用的 DTO，用 record 一行搞定。建立 model/BookDto.java：

package model;

public record BookDto(
    String title,
    String author,
    String isbn,
    int totalCopies
) {
    // record 可以加驗證邏輯（compact constructor）
    public BookDto {
        if (title == null || title.isBlank()) {
            throw new IllegalArgumentException("書名不能為空");
        }
        if (totalCopies < 0) {
            throw new IllegalArgumentException("數量不能為負數");
        }
    }

    // record 也可以加自訂方法
    public String displayName() {
        return title + " — " + author;
    }
}

執行測試：在 Main.java 加一段，確認 record 可用：

import model.BookDto;

public class Main {
    public static void main(String[] args) {
        // 測試 record
        var book = new BookDto("Effective Java", "Joshua Bloch", "978-0134685991", 3);
        System.out.println(book.displayName());
        System.out.println(book);  // 自動產生的 toString()
    }
}

Step 2：建立 BaseEntity（Abstract Class）

所有 Entity 共用的 id 和時間戳記。這是 Spring Boot 專案中 @MappedSuperclass 的原始形態。建立 model/BaseEntity.java：

package model;

import java.time.LocalDateTime;
import java.util.UUID;

public abstract class BaseEntity {
    private final String id;
    private final LocalDateTime createdAt;

    // 建構子 —— 子類別透過 super() 呼叫
    protected BaseEntity() {
        this.id = UUID.randomUUID().toString();
        this.createdAt = LocalDateTime.now();
    }

    // 具體方法 —— 所有子類別直接繼承
    public String getId() {
        return id;
    }

    public LocalDateTime getCreatedAt() {
        return createdAt;
    }

    // 抽象方法 —— 子類別必須實作
    public abstract String getTypeLabel();

    // final 方法 —— 子類別不能覆寫
    public final String getEntityInfo() {
        return "[" + getTypeLabel() + "] " + id + " (created: " + createdAt + ")";
    }
}
💡 為什麼用 protected 建構子？ protected 表示只有子類別可以呼叫這個建構子（透過 super()）。外部程式碼不能直接 new BaseEntity()。這是 Java 中控制抽象類別實例化的標準做法。

Step 3：建立 LibraryItem（Interface）

定義「可被借閱」的合約。建立 model/LibraryItem.java：

package model;

public interface LibraryItem {
    // 抽象方法 —— 實作類別必須提供
    boolean borrow(String userId);
    boolean returnItem(String userId);

    // default 方法 —— 提供預設實作
    default boolean isAvailable() {
        return getAvailableCount() > 0;
    }

    // 另一個抽象方法
    int getAvailableCount();

    // static 方法 —— 工廠方法
    static String formatDueDate(int days) {
        return "到期日：" + java.time.LocalDate.now().plusDays(days);
    }
}

注意到 isAvailable() 是一個 default 方法——它依賴 getAvailableCount() 這個抽象方法。這就是 T02 提到的 interface default 方法的典型用法：用抽象方法組合出高階的具體方法。Spring Data JPA 正是這樣做的——它從你的方法名稱推斷查詢邏輯。

Step 4：建立 PhysicalBook 和 EBook

建立 entity/PhysicalBook.java：

package entity;

import model.BaseEntity;
import model.LibraryItem;

public class PhysicalBook extends BaseEntity implements LibraryItem {
    private final String title;
    private final String isbn;
    private int totalCopies;      // 總館藏數
    private int borrowedCopies;   // 已借出數

    public PhysicalBook(String title, String isbn, int totalCopies) {
        super();  // 呼叫 BaseEntity 建構子，產生 id 和 createdAt
        this.title = title;
        this.isbn = isbn;
        this.totalCopies = totalCopies;
        this.borrowedCopies = 0;
    }

    // === 實作 LibraryItem ===
    @Override
    public boolean borrow(String userId) {
        if (borrowedCopies < totalCopies) {
            borrowedCopies++;
            System.out.println(userId + " 借了實體書：" + title);
            return true;
        }
        System.out.println("實體書 " + title + " 已全數借出");
        return false;
    }

    @Override
    public boolean returnItem(String userId) {
        if (borrowedCopies > 0) {
            borrowedCopies--;
            System.out.println(userId + " 歸還了實體書：" + title);
            return true;
        }
        return false;
    }

    @Override
    public int getAvailableCount() {
        return totalCopies - borrowedCopies;
    }

    // === 實作 BaseEntity 的抽象方法 ===
    @Override
    public String getTypeLabel() {
        return "實體書";
    }
}

建立 entity/EBook.java：

package entity;

import model.BaseEntity;
import model.LibraryItem;

public class EBook extends BaseEntity implements LibraryItem {
    private final String title;
    private final double fileSizeMb;
    private boolean isBorrowed;  // 電子書同一時間只能一人借

    public EBook(String title, double fileSizeMb) {
        super();
        this.title = title;
        this.fileSizeMb = fileSizeMb;
        this.isBorrowed = false;
    }

    // === 實作 LibraryItem ===
    @Override
    public boolean borrow(String userId) {
        if (!isBorrowed) {
            isBorrowed = true;
            System.out.println(userId + " 借了電子書：" + title + " (" + fileSizeMb + "MB)");
            return true;
        }
        System.out.println("電子書 " + title + " 已被借出");
        return false;
    }

    @Override
    public boolean returnItem(String userId) {
        if (isBorrowed) {
            isBorrowed = false;
            System.out.println(userId + " 歸還了電子書：" + title);
            return true;
        }
        return false;
    }

    @Override
    public int getAvailableCount() {
        return isBorrowed ? 0 : 1;
    }

    @Override
    public String getTypeLabel() {
        return "電子書";
    }
}

Step 5：建立 LibraryService

這是 Spring Boot 中 @Service 的原形。建立 service/LibraryService.java：

package service;

import entity.PhysicalBook;
import entity.EBook;
import model.LibraryItem;

import java.util.ArrayList;
import java.util.List;

public class LibraryService {
    // 針對介面寫程式（Program to Interface）
    private final List<LibraryItem> items = new ArrayList<>();

    // 方法多載（overloading）：兩種新增方式
    public void addItem(PhysicalBook book) {
        items.add(book);
        System.out.println("新增實體書：" + book.getEntityInfo());
    }

    public void addItem(EBook book) {
        items.add(book);
        System.out.println("新增電子書：" + book.getEntityInfo());
    }

    public void borrowItem(int index, String userId) {
        if (index < 0 || index >= items.size()) {
            System.out.println("索引超出範圍");
            return;
        }
        items.get(index).borrow(userId);
    }

    public void listAllItems() {
        System.out.println("\n===== 館藏目錄 =====");
        for (int i = 0; i < items.size(); i++) {
            LibraryItem item = items.get(i);
            System.out.println(
                "[" + i + "] " +
                item.getEntityInfo() +
                " | 可借數量：" + item.getAvailableCount()
            );
        }
        System.out.println("===== 總計 " + items.size() + " 筆 =====\n");
    }
}
🎯 觀察重點 List<LibraryItem> —— 我們針對 interface 而非具體類別來寫邏輯。listAllItems() 完全不知道也不在乎它處理的是 PhysicalBook 還是 EBook，這就是「針對介面寫程式」的威力。Spring Boot 的 DI 容器正是建立在這個原則之上。

Step 6：在 Main.java 串起一切

把 Main.java 改寫為以下內容：

import model.BookDto;
import entity.PhysicalBook;
import entity.EBook;
import service.LibraryService;

public class Main {
    public static void main(String[] args) {
        // === DTO 測試 ===
        var dto = new BookDto("Clean Code", "Robert C. Martin", "978-0132350884", 5);
        System.out.println("📖 " + dto.displayName());

        // === 建立 Service ===
        var library = new LibraryService();

        // === 加入館藏（方法多載） ===
        library.addItem(new PhysicalBook("Effective Java", "978-0134685991", 3));
        library.addItem(new PhysicalBook("Design Patterns", "978-0201633610", 2));
        library.addItem(new EBook("Spring in Action", 8.5));

        // === 查看全部 ===
        library.listAllItems();

        // === 借閱測試 ===
        System.out.println("\n--- 借閱操作 ---");
        library.borrowItem(0, "user001");  // 借 Effective Java
        library.borrowItem(0, "user002");  // 借 Effective Java（還有庫存）
        library.borrowItem(0, "user003");  // 借 Effective Java（最後一本）
        library.borrowItem(0, "user004");  // 借不到！
        library.borrowItem(2, "user001");  // 借電子書

        // === 再次查看 ===
        library.listAllItems();

        // === Interface default 方法測試 ===
        System.out.println("到期日資訊：" +
            model.LibraryItem.formatDueDate(14));
    }
}

課後檢查清單

  • BookDto record 成功建立，含驗證邏輯和自訂方法
  • BaseEntity abstract class 含具體方法 + 抽象方法 + final 方法
  • LibraryItem interface 含抽象方法 + default 方法 + static 方法
  • PhysicalBook / EBook 各自以不同邏輯實作 borrow/returnItem
  • LibraryService 使用 List<LibraryItem>（針對介面寫程式）
  • 方法多載（addItem）正確區分不同參數型別
  • Main.java 執行後輸出符合預期
  • 理解 package 目錄結構（model / entity / service）將來就是 Spring Boot 的分層

📌 這堂課建立的 OOP 心智模型

record→ 等於 Spring Boot 的 DTO / Request / Response
abstract class→ 等於 Spring Boot 的 @MappedSuperclass / BaseEntity
interface→ 等於 Spring Boot 的 Service interface / Repository
List<Interface>→ DI 容器的本質——針對介面而非實作寫程式
