第一個 Spring Boot 應用：從 start.spring.io 到 live API

這堂課你會建立真正的 Spring Boot 專案，寫出第一個 REST Controller，並親眼看到 T04 中提到的 IoC 容器如何自動組裝一切。

🎯 這堂課的目標 建立一個 Spring Boot 專案 → 寫一個 Controller 包含 GET 和 POST → 用瀏覽器和 IntelliJ 測試 → 理解 Spring Boot 的啟動過程。完成後你就跨過了從「純 Java」到「Java 後端框架」的分界線。

Step 1：用 Spring Initializr 建立專案

Spring 官方提供了一個網頁工具來產生專案骨架。有兩種方式：

方式 A：從 IntelliJ 內建（推薦）

  1. IntelliJ → New Project
  2. 左側選 Spring Boot（IntelliJ 內建了 Spring Initializr）
  3. 右側設定：
    Name：demo
    Language：Java
    Type：Gradle
    Group：com.example
    JDK：21
  4. Next → 勾選依賴（Dependencies）：Spring Web
  5. Create

方式 B：從 start.spring.io 下載

  1. 開啟 start.spring.io
  2. Project: Gradle、Language: Java、Spring Boot: 3.4.x、Group: com.example
  3. Dependencies: 加入 Spring Web
  4. Generate → 下載 ZIP → 解壓縮 → IntelliJ 開啟資料夾
⚠️ 首次使用 Gradle 注意 這是課程中第一次使用 Gradle（P01 用的是 IntelliJ build system）。Gradle 是 Java 生態系的標準建置工具，負責管理依賴、編譯、測試、打包。IntelliJ 會在專案開啟後自動下載 Gradle 和依賴——第一次可能需要 3-5 分鐘，請耐心等待右下角的進度條跑完。看到 BUILD SUCCESSFUL 才算就緒。

Step 2：認識專案結構

Gradle 專案比之前的 java-practice 多了一些檔案：

demo/
├── build.gradle                    ← 依賴宣告（等於 package.json + tsconfig）
├── settings.gradle                 ← 專案名稱
├── gradlew / gradlew.bat           ← Gradle wrapper（免安裝 Gradle）
├── src/
│   ├── main/
│   │   ├── java/com/example/demo/
│   │   │   └── DemoApplication.java  ← 啟動類別！
│   │   └── resources/
│   │       ├── application.properties ← Spring Boot 設定檔
│   │       ├── static/                ← 靜態資源（HTML/CSS/JS）
│   │       └── templates/             ← 模板（Thymeleaf，本課暫不用）
│   └── test/
│       └── java/com/example/demo/
│           └── DemoApplicationTests.java

先打開 DemoApplication.java，認識這個最重要的檔案：

package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication   // ← 這一行觸發了整個 IoC 容器的啟動
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
        // ↑ 這一行做了 T04 說的六件事：掃描、註冊 Bean、注入、自動配置、啟動 Tomcat
    }
}

再打開 build.gradle，認識依賴管理：

plugins {
    id 'java'
    id 'org.springframework.boot' version '3.4.x'
    id 'io.spring.dependency-management' version '1.1.x'
}

repositories {
    mavenCentral()       // 等於 npm registry
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    // ↑ 等於 npm install express —— 引入 Spring Web（內含 Tomcat + Spring MVC）
    testImplementation 'org.springframework.boot:spring-boot-starter-test'
}
💡 Gradle 對照 npm
build.gradle↔ package.json
implementation↔ dependencies
mavenCentral()↔ npm registry
gradlew build↔ npm run build
gradlew test↔ npm test

Step 3：第一個 GET Endpoint

建立 controller/HelloController.java：

package com.example.demo.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController   // ← Spring 自動發現並註冊為 Bean
public class HelloController {

    @GetMapping("/hello")   // ← 對應 GET http://localhost:8080/hello
    public String sayHello() {
        return "Hello, Spring Boot! 🌱";
    }
}

啟動！在 DemoApplication.java 的 main 方法旁點 ▶，或按 Ctrl+Shift+F10。

觀察 IntelliJ 下方 Run 視窗的輸出——你應該會看到 Spring Boot 的標誌和這些關鍵訊息：

Step 4：加入 POST Endpoint（接收 JSON）

建立 model/GreetingRequest.java（用 record）：

package com.example.demo.model;

public record GreetingRequest(String name, String language) { }

更新 HelloController.java，加入 POST：

package com.example.demo.controller;

import com.example.demo.model.GreetingRequest;
import org.springframework.web.bind.annotation.*;

@RestController
public class HelloController {

    @GetMapping("/hello")
    public String sayHello() {
        return "Hello, Spring Boot! 🌱";
    }

    // === 新增：POST endpoint ===
    @PostMapping("/hello")               // ← 對應 POST
    public String greet(@RequestBody GreetingRequest request) {
        // @RequestBody → 等於 FastAPI 中讀取 request body（自動 JSON 反序列化）
        String greeting = switch (request.language()) {
            case "zh" -> "你好";
            case "ja" -> "こんにちは";
            default   -> "Hello";
        };
        return greeting + ", " + request.name() + "!";
    }
}
💡 停掉舊的再重跑 改完程式碼後，先按紅色方塊 ■ 停止上次的執行，再重新 ▶ 啟動。Spring Boot 有 DevTools 可以熱重載，但現在先手動重啟就好。

測試 POST

用 PowerShell 測試（開一個新的終端機視窗）：

# Windows PowerShell:
Invoke-RestMethod -Uri http://localhost:8080/hello -Method Post -Body '{"name":"Alice","language":"zh"}' -ContentType "application/json"

# 或使用 curl（PowerShell 5+ 內建）：
curl -X POST http://localhost:8080/hello -H "Content-Type: application/json" -d '{"name":"Alice", "language":"ja"}'

預期輸出：你好, Alice! 或 こんにちは, Alice!

Step 5：加入 Service 層（親眼見證 DI）

目前 Controller 直接回傳字串，沒有業務邏輯層。現在加入 Service，見證 Spring 如何自動注入依賴。

建立 service/GreetingService.java：

package com.example.demo.service;

import org.springframework.stereotype.Service;
import java.time.LocalTime;

@Service   // ← 告訴 Spring：「這是一個 Service Bean，請管理它」
public class GreetingService {

    public String createGreeting(String name, String language) {
        String greeting = switch (language) {
            case "zh" -> "你好";
            case "ja" -> "こんにちは";
            default   -> "Hello";
        };

        int hour = LocalTime.now().getHour();
        String timePrefix = hour < 12 ? "早安！" : "午安！";

        return timePrefix + greeting + ", " + name + "!";
    }
}

改寫 HelloController.java，透過建構子注入 Service：

package com.example.demo.controller;

import com.example.demo.model.GreetingRequest;
import com.example.demo.service.GreetingService;
import org.springframework.web.bind.annotation.*;

@RestController
public class HelloController {

    private final GreetingService greetingService;

    // 建構子注入：Spring 會自動從 IoC 容器中找到 GreetingService 並傳入
    public HelloController(GreetingService greetingService) {
        this.greetingService = greetingService;
    }

    @GetMapping("/hello")
    public String sayHello() {
        return greetingService.createGreeting("World", "en");
    }

    @PostMapping("/hello")
    public String greet(@RequestBody GreetingRequest request) {
        return greetingService.createGreeting(request.name(), request.language());
    }
}
🔥 你剛剛見證了 Dependency Injection
  1. 你在 GreetingService 上加了 @Service → Spring 掃描到它，建立一個實例放入 IoC 容器
  2. 你在 HelloController 的建構子中宣告需要 GreetingService → Spring 自動從容器中找出並傳入
  3. 你沒有寫過一行 new GreetingService()——整個過程由框架自動完成
這就是 T04 說的控制反轉。下一堂課 T05 會深入它的底層機制。

Step 6：觀察 IoC 容器（可選但推薦）

在 DemoApplication.java 的 main 方法中加一段，列印所有被 Spring 管理的 Bean：

@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        var ctx = SpringApplication.run(DemoApplication.class, args);

        // 印出所有 Bean 的名稱
        System.out.println("\n====== Spring IoC 容器中的 Beans ======");
        var beanNames = ctx.getBeanDefinitionNames();
        for (String name : beanNames) {
            System.out.println("  🫘 " + name);
        }
        System.out.println("總數：" + beanNames.length + " 個 Beans\n");
    }
}

你會看到幾十個 Bean——大部分是 Spring 內建的（Tomcat、JSON parser 等），也會看到你自己的 helloController 和 greetingService。

📌 這堂課建立的 Spring Boot 心智模型

@SpringBootApplication啟動類別，觸發掃描 + 自動配置 + 內嵌 Tomcat
@RestControllerHTTP Controller，自動註冊路由 + 回傳 JSON
@GetMapping / @PostMapping註冊 HTTP 路由（對應 FastAPI 的 decorator）
@RequestBody自動將 JSON body 反序列化為 Java 物件
@Service + 建構子宣告 Bean + 自動注入依賴（DI）
GradleJava 的 npm——管理依賴、編譯、打包
