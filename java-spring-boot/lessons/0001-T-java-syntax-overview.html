1. 型別系統：宣告在左，型別在前

三種語言的變數宣告放在一起看：

JavaTypeScriptPython
String name = "Alice"; let name: string = "Alice"; name: str = "Alice"

核心差異：

  • Java 把型別寫在變數前面，不是後面（`String name` vs `name: string`）
  • Java 是強靜態型別——型別在編譯期就鎖死，不像 Python 可以隨時換型別
  • 如果你用過 TypeScript 的 strict mode，那個感覺很接近——只是 Java 沒有 `any` 可以逃
💡 順帶一提：Java 的 var Java 10 引入了 var 關鍵字做區域變數型別推斷。你可以寫 var name = "Alice";，編譯器自動推斷為 String。但和 JS 的 var 完全不同——它仍然是強型別，推斷後就不能換型別。在 Spring Boot 程式碼中你會經常看到 var。

2. 基本型別 vs 物件型別：大小寫有差

Java 把資料分成兩大陣營。這是 TS/Python 開發者最容易踩的坑：

🔹 基本型別（Primitives）

小寫開頭，存的是「值本身」

  • int — 整數
  • double — 浮點數
  • boolean — true/false
  • char — 單一字元
  • long — 長整數

效能高，不可為 null

🔸 物件型別（Objects）

大寫開頭，存的是「參考（reference）」

  • String — 字串（注意大寫！）
  • Integer — 整數的包裝
  • Double — 浮點數的包裝
  • Boolean — 布林的包裝
  • List<T> — 列表

可以為 null，有方法可呼叫

實務上在 Spring Boot 中，Entity 的欄位通常用物件型別（如 Long id 而不是 long id），因為資料庫欄位可能為 null。

⚠️ 常見陷阱 String 不是基本型別！它是正牌的 class。所以你可以寫 "hello".length()。另外，int 不能放進 List<int>，必須寫 List<Integer>。

3. 類別與方法：一切都在 class 裡

Java 是純物件導向語言。沒有游離在 class 外的函式或變數。每個 .java 檔案通常包含一個 public class，且檔名必須和 public class 名稱一致。

以一個 Spring Boot 中常見的 Service 類別為例：

// 檔名必須是 UserService.java
public class UserService {

    // 屬性 (fields) —— 通常宣告為 private
    private String defaultStatus;

    // 建構子 (constructor) —— 名稱和 class 一樣，沒有回傳型別
    public UserService(String defaultStatus) {
        this.defaultStatus = defaultStatus;
    }

    // 方法 —— 回傳型別寫在方法名稱前面
    public String getStatus(Long userId) {
        return this.defaultStatus;
    }

    // void 表示不回傳任何值
    public void setDefaultStatus(String status) {
        this.defaultStatus = status;
    }
}

💡 順帶一提：封裝與 Getter/Setter 在 Java 中，屬性通常宣告為 private，並提供 public 的 getter/setter 方法來存取。這和 Python/JS 可以直接 obj.field = value 的文化不同。Spring Boot 大量依賴這個慣例——框架透過 getter/setter 來完成 DI（依賴注入）和序列化。在 IntelliJ 中，你可以用 Alt+Insert 自動生成它們。

4. 陣列 vs 列表：固定 vs 可變

這是 TS/Python 開發者最大的認知轉換之一。

在 JS 和 Python 中，Array/List 天生就是動態大小的。但在 Java 中：

// ❌ 原生陣列：大小在建立時就鎖死，永遠不能改變！
String[] items = new String[3];
items[0] = "Sword";
// items[3] = "Shield"; // → ArrayIndexOutOfBoundsException！

如果你要可以動態增減的容器（就像 JS Array 或 Python list），必須用 ArrayList：

import java.util.List;
import java.util.ArrayList;

// ✅ 動態列表 —— 這才是你在 Spring Boot 中幾乎永遠會用的
List<String> items = new ArrayList<>();
items.add("Sword");      // 相當於 JS 的 push() / Python 的 append()
items.add("Shield");
items.add("Potion");

String first = items.get(0);   // 相當於 items[0]
int count = items.size();      // 相當於 items.length / len(items)
items.remove(1);               // 移除 index 1 的元素

注意 List（介面）vs ArrayList（實作）的宣告習慣——這是 Java 「針對介面寫程式」的慣例，和 TS 的 interface 思維類似。Spring Boot 的所有集合型返回值也都是宣告為 List<T>。

5. 控制流程：與 JS 幾乎一樣

好消息：Java 的大括號、if/else、for/while 語法和 JS 幾乎完全相同——因為 JS 的語法本來就是從 Java 借來的。

// if-else —— 和 JS 一模一樣
if (score >= 60) {
    System.out.println("Pass");
} else {
    System.out.println("Fail");
}

// for-each —— Java 用冒號（:），不是 of/in
for (String item : items) {
    System.out.println(item);
}

// 傳統 for —— 和 JS/C 完全一樣
for (int i = 0; i < items.size(); i++) {
    System.out.println(i + ": " + items.get(i));
}

// while —— 一樣
while (items.size() < 10) {
    items.add("filler");
}

6. 字串比較：用 equals()，不是 ==

頭號新手陷阱。在 Java 中：

String a = "hello";
String b = new String("hello");

a == b;          // false！因為 == 比較的是「記憶體參考」，不是內容
a.equals(b);     // true  —— 這才是比較字串內容的正確方法

這是因為 String 是物件型別，== 對物件比較的是參考位址。Spring Boot 程式碼中到處都是 .equals()。

7. 進入點：main 方法

每個 Java 程式都需要一個起點。Spring Boot 會自動生成它，但你需要認識它的樣子：

public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, Spring Boot!");
    }
}
  • public static void main(String[] args) — 這個簽章是固定的，一個字都不能改
  • System.out.println() — Java 版的 console.log()
  • static 表示這是類別層級的方法，不需要先 new 物件就能呼叫

試著不看答案，在心裡回答以下問題。這是 retrieval practice——自己從記憶中提取答案，比直接重讀更有效。

Q1: Java 中哪兩種型別類別的大小寫規則不同？為什麼 int 不能是 null 但 Integer 可以？

點擊展開答案

Primitives（小寫：int, double, boolean）存的是值本身，所以不能為 null。物件型別（大寫：Integer, String）存的是參考（reference），可以指向 null。在 Spring Boot 的 Entity 欄位中通常用 Long、Integer 等包裝型別，因為資料庫欄位可能為空。

Q2: 你想建立一個可以動態增減的字串列表，該用 String[] 還是 ArrayList<String>？為什麼？

點擊展開答案

用 ArrayList<String>（通常宣告為 List<String>）。Java 原生陣列 String[] 在建立後大小固定，無法 add() 或 remove()。Spring Boot 中處理集合幾乎都用 List。

Q3: Java 中比較兩個字串的內容，該用 == 還是 .equals()？

點擊展開答案

用 .equals()。== 比較的是物件的記憶體參考（reference），不是內容。兩個內容相同但不同實例的字串，== 會回傳 false。
