面試重點總複習

這堂課將 22 堂課中的面試關鍵知識濃縮為七大主題，每個主題附標準回答框架。適合面試前快速翻閱、在通勤時最後衝刺。

🎯 使用方式 每個主題包含：核心概念（30 秒版本）→ 面試常見追問 → 關鍵程式碼。先試著自己回答，再對照答案。

🔥 1. DI 與 IOC（T05）

面試題：「請解釋 IoC 和 DI」

標準回答：IoC（控制反轉）是設計原則——把物件建立和管理的控制權從程式碼移交給框架。DI（依賴注入）是實現 IoC 的具體手段——框架自動將依賴注入到需要的地方。Spring 的 ApplicationContext 在啟動時掃描所有 @Component/@Service/@Repository/@Controller，建立 singleton Bean 放入容器，並透過建構子注入自動組裝依賴圖。

常見追問你的回答
三種注入方式？建構子注入（✅ final、易測試）、Setter 注入（選擇性依賴）、欄位注入（❌ 難測試）
同介面多實作？@Primary 設預設 + @Qualifier("beanName") 指名
Spring 如何知道要注入什麼？根據建構子參數的型別從容器中匹配，多個同型別時用 bean 名稱
FastAPI Depends 和 Spring DI 的差異？FastAPI 是請求範圍（每次請求解析），Spring 是應用範圍（singleton 預建）

🔥 2. Bean 生命週期（T06）

面試題：「描述 Spring Bean 的生命週期」

標準回答：實例化（建構子）→ 填充屬性（注入 @Autowired、@Value）→ BeanPostProcessor:before → @PostConstruct 初始化 → BeanPostProcessor:after（AOP proxy 在此建立）→ 就緒 → @PreDestroy 銷毀。關鍵點：AOP 代理在 @PostConstruct 之後才建立，所以 @PostConstruct 中呼叫 @Transactional 方法不會生效。

常見追問你的回答
Singleton vs Prototype？Singleton 全容器一個實例（預設），Prototype 每次注入/查詢新建。Prototype 的 @PreDestroy 不會被呼叫
Singleton 注入 Prototype 的陷阱？Prototype 只被注入一次（隨 singleton 建立時），之後都是同一個實例。解法：ObjectProvider 或 @Lookup
BeanPostProcessor 是什麼？Spring 的萬能中介層，在每個 Bean 初始化前後插入邏輯。AOP、@Transactional、@Async 都是靠它實現

🔥 3. Java Stream API（T03）

面試題：「用 Stream 對 List<Employee> 找出各部門薪資最高的三人」

現場寫法：

Map<String, List<Employee>> topByDept = employees.stream()
    .collect(Collectors.groupingBy(Employee::department,
        Collectors.collectingAndThen(Collectors.toList(),
            list -> list.stream()
                .sorted(Comparator.comparingInt(Employee::salary).reversed())
                .limit(3).toList())));
常見追問你的回答
中間操作 vs 終端操作？中間操作（filter/map/sorted）回傳 Stream，lazy；終端操作（collect/reduce/count）觸發執行並關閉 Stream
flatMap 做什麼？攤平巢狀結構：把 Stream<List<T>> 變成 Stream<T>。例如攤平所有員工的所有技能
mapToInt 為什麼存在？避免 int ↔ Integer 的自動裝箱，提升效能。MapToInt 回傳 IntStream，有 sum()/average()
Collectors.groupingBy？等於 SQL GROUP BY。搭配 counting()/averagingInt()/mapping() 做聚合

🔥 4. 執行緒安全（T08）

面試題：「Spring Boot 的 Service 是執行緒安全的嗎？」

標準回答：預設 singleton 的 Service本身不是執行緒安全的——但可以藉由設計來保證安全。關鍵原則是無狀態（stateless）：Service 不持有可變的實例欄位。所有狀態存在資料庫或外部快取中。如果有必須持有的共享可變狀態，用 ConcurrentHashMap、AtomicInteger 等並行安全容器。Spring 本身不提供執行緒安全保證——這是開發者的責任。

常見追問你的回答
HashMap 在 singleton 中有什麼問題？多執行緒同時 put 可能導致死循環（JDK 7）或資料遺失。用 ConcurrentHashMap
樂觀鎖 vs 悲觀鎖？樂觀：@Version 版本號比對 + 重試，適合低衝突。悲觀：SELECT FOR UPDATE 鎖定行，適合高衝突（秒殺）
@Transactional 自呼叫為什麼失效？AOP proxy 只攔截外部呼叫。同 class 內的 this.method() 繞過 proxy。解法：抽到另一個 Service
交易隔離級別？READ_COMMITTED（預設，防 dirty read）。SERIALIZABLE（最安全但最慢）。面試說得出四級即可

🔥 5. JPA 與 N+1 問題（T07 + P12）

面試題：「什麼是 N+1 查詢問題？如何解決？」

標準回答：N+1 發生在 ORM 延遲載入（LAZY）時：先查主表（1 條 SQL），然後對每一筆結果的關聯欄位各發一條 SQL（N 條）。例如查 10 篇文章 → 1 條查文章 + 10 條查作者 = 11 條 SQL。解法：① @EntityGraph 指定要 JOIN 的關聯 ② JPQL 中用 JOIN FETCH ③ @NamedEntityGraph。核心思想：一條 SQL 載入所有需要的關聯。

常見追問你的回答
@OneToMany 和 @ManyToOne 的 fetch 預設？@OneToMany LAZY（好）；@ManyToOne EAGER（不好，建議手動改 LAZY）
JPA / Hibernate / Spring Data JPA 關係？JPA 是規格，Hibernate 是實作，Spring Data JPA 是便利層（自動 Repository）
衍生查詢方法？findBy + 屬性 + 條件 + 排序。例如 findByEmailContainingOrderByCreatedAtDesc
Entity 和 DTO 為何分開？Entity 對應資料表（含 JPA 註解），DTO 對應 API 傳輸（控制暴露欄位、不含密碼）

🔥 6. 三層架構與 REST 設計（T04 + T10）

面試題：「描述 Spring Boot 的標準分層架構」

標準回答：Controller（@RestController）：接收 HTTP、參數驗證、回傳回應，不寫業務邏輯。Service（@Service）：業務邏輯、交易管理、協調 Repository，不知道 HTTP 存在。Repository（JpaRepository interface）：資料存取，Spring Data 自動產生實作。依賴方向是單向的：Controller → Service → Repository。

常見追問你的回答
為什麼需要 Service interface？鬆耦合、易測試、易替換實作。小專案可直接用 class，但面試中推崇 interface
@ControllerAdvice 做什麼？集中管理所有 Controller 的例外處理，統一錯誤格式
201 vs 200 vs 204？200 一般成功、201 POST 建立成功、204 DELETE 成功無 body
如何處理 API 版本？URL 前綴 /api/v1/（最常見）、Header（乾淨但隱晦）、Query param

🔥 7. Java 基礎 — 高頻陷阱題

陷阱題你的回答
String 比較用 == 還是 equals？equals()。== 比較的是記憶體參考，不是內容
String 為什麼是 final？安全性（不可變 = 執行緒安全）+ String pool 效能 + HashMap key 可靠性
ArrayList vs LinkedList？ArrayList O(1) 隨機存取（動態陣列）；LinkedList O(1) 頭尾插入（雙向鏈結）。實務中 ArrayList 佔 95%
int vs Integer？int 是 primitive（不可 null、效能高）；Integer 是 wrapper（可 null、可用於泛型）
Java record 是什麼？JDK 14+，自動產生 constructor/getter/equals/hashCode/toString 的不可變資料載體。Spring Boot 中用作 DTO
final 的三種用法？final 變數（不可重賦值）、final 方法（不可覆寫）、final class（不可繼承）
Checked vs Unchecked Exception？Checked：編譯器強制 try-catch 或 throws（IOException）。Unchecked：RuntimeException 子類，不強制處理

🧪 白板題速練

白板 1：寫一個 Stream 管線，從 List<Order> 中篩出金額 > 1000、依日期排序、取前 5 筆、收集訂單 ID

答案 
var ids = orders.stream()
    .filter(o -> o.amount() > 1000)
    .sorted(Comparator.comparing(Order::date))
    .limit(5)
    .map(Order::id)
    .toList();

白板 2：用 @ControllerAdvice 處理 MethodArgumentNotValidException，回傳欄位名稱和錯誤訊息

答案 
@ExceptionHandler(MethodArgumentNotValidException.class)
@ResponseStatus(HttpStatus.BAD_REQUEST)
public ApiResponse<Void> handle(MethodArgumentNotValidException ex) {
    var errors = ex.getBindingResult().getFieldErrors().stream()
        .map(e -> e.getField() + ": " + e.getDefaultMessage())
        .toList();
    return ApiResponse.error("VALIDATION", String.join("; ", errors));
}

白板 3：寫一個 @EntityGraph 解決 Post 查詢時的 author + category N+1 問題

答案 
@EntityGraph(attributePaths = {"author", "category"})
Page<Post> findByStatus(PostStatus status, Pageable pageable);
