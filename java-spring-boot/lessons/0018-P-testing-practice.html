JUnit + MockMvc 測試實作

這堂課會為 P07 的 Task 系統寫出三層測試，覆蓋 T09 學到的所有測試層級。你的 demo 專案已經有 spring-boot-starter-test（Gradle 建立時就自動包含了），不需要額外安裝。

🎯 這堂課的目標 Service 單元測試（Mockito）→ Controller 測試（@WebMvcTest + MockMvc）→ Repository 測試（@DataJpaTest）→ 一則整合測試（@SpringBootTest）。全部用 IntelliJ 執行，看到綠色 ✓。

測試目錄結構

src/test/java/com/example/demo/
├── service/
│   └── TaskServiceTest.java        ← 單元測試（Mockito）
├── controller/
│   └── TaskControllerTest.java     ← Web 層測試（MockMvc）
├── repository/
│   └── TaskRepositoryTest.java     ← JPA 測試（@DataJpaTest）
└── DemoApplicationTests.java       ← 整合測試（已存在，改寫它）
💡 在 IntelliJ 中建立測試 打開被測的 class（例如 TaskService.java）→ 按 Ctrl+Shift+T → Create New Test → 勾選你想測的方法 → OK。IntelliJ 會在對應的 test 目錄下自動建立測試骨架。

Step 1：Service 單元測試（Mockito）

建立 src/test/java/.../service/TaskServiceTest.java：

package com.example.demo.service;

import com.example.demo.model.Task;
import com.example.demo.repository.TaskRepository;
import org.junit.jupiter.api.*;
import org.junit.jupiter.api.extension.ExtendWith;
import org.mockito.*;
import org.mockito.junit.jupiter.MockitoExtension;
import java.util.*;

import static org.assertj.core.api.Assertions.*;
import static org.mockito.Mockito.*;

@ExtendWith(MockitoExtension.class)
class TaskServiceTest {

    @Mock
    private TaskRepository taskRepo;

    @InjectMocks
    private TaskServiceImpl taskService;

    // === getAllTasks ===
    @Test
    void shouldReturnAllTasks() {
        var tasks = List.of(new Task("A"), new Task("B"));
        when(taskRepo.findAll()).thenReturn(tasks);

        var result = taskService.getAllTasks();

        assertThat(result).hasSize(2);
        verify(taskRepo).findAll();  // 確認有呼叫到
    }

    // === getTask ===
    @Test
    void shouldReturnTaskWhenExists() {
        var task = new Task("Test");
        when(taskRepo.findById(1L)).thenReturn(Optional.of(task));

        var result = taskService.getTask(1L);

        assertThat(result).isPresent();
        assertThat(result.get().getTitle()).isEqualTo("Test");
    }

    @Test
    void shouldReturnEmptyWhenNotFound() {
        when(taskRepo.findById(999L)).thenReturn(Optional.empty());

        var result = taskService.getTask(999L);

        assertThat(result).isEmpty();
    }

    // === updateTask ===
    @Test
    void shouldUpdateTaskTitle() {
        var task = new Task("Old");
        when(taskRepo.findById(1L)).thenReturn(Optional.of(task));
        when(taskRepo.save(any())).thenReturn(task);

        var result = taskService.updateTask(1L, "New", null);

        assertThat(result.getTitle()).isEqualTo("New");
        verify(taskRepo).save(task);
    }

    // === deleteTask ===
    @Test
    void shouldDeleteTask() {
        doNothing().when(taskRepo).deleteById(1L);

        taskService.deleteTask(1L);

        verify(taskRepo).deleteById(1L);
    }
}
🔍 Mockito 快速對照
when(x.y()).thenReturn(z)設定 mock 的回傳值
verify(x).y()驗證 mock 的方法有被呼叫
verify(x, never()).y()驗證 mock 的方法從未被呼叫
any() / anyLong()匹配任意參數
doNothing().when(x).y()mock void 方法
doThrow(...).when(x).y()mock 方法拋出例外

Step 2：Controller 測試（@WebMvcTest）

建立 controller/TaskControllerTest.java：

package com.example.demo.controller;

import com.example.demo.model.Task;
import com.example.demo.service.TaskService;
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.autoconfigure.web.servlet.WebMvcTest;
import org.springframework.http.MediaType;
import org.springframework.test.context.bean.override.mockito.MockitoBean;
import org.springframework.test.web.servlet.MockMvc;
import java.util.*;

import static org.mockito.Mockito.*;
import static org.springframework.test.web.servlet.request.MockMvcRequestBuilders.*;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.*;

@WebMvcTest(TaskController.class)
class TaskControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockitoBean       // ← Spring Boot 3.4+（舊版用 @MockBean）
    private TaskService taskService;

    @Test
    void shouldReturnAllTasks() throws Exception {
        var tasks = List.of(new Task("A"), new Task("B"));
        when(taskService.getAllTasks()).thenReturn(tasks);

        mockMvc.perform(get("/api/tasks"))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.length()").value(2))
            .andExpect(jsonPath("$[0].title").value("A"));
    }

    @Test
    void shouldReturn404WhenTaskNotFound() throws Exception {
        when(taskService.getTask(999L)).thenReturn(Optional.empty());

        mockMvc.perform(get("/api/tasks/999"))
            .andExpect(status().isNotFound());
    }

    @Test
    void shouldCreateTask() throws Exception {
        var task = new Task("New Task");
        when(taskService.createTask(eq("New Task"), any()))
            .thenReturn(task);

        mockMvc.perform(post("/api/tasks")
                .contentType(MediaType.APPLICATION_JSON)
                .content("""
                    {"title":"New Task","category":"Work"}
                    """))
            .andExpect(status().isCreated())
            .andExpect(jsonPath("$.title").value("New Task"));
    }

    @Test
    void shouldRejectEmptyTitle() throws Exception {
        mockMvc.perform(post("/api/tasks")
                .contentType(MediaType.APPLICATION_JSON)
                .content("""
                    {"title":"","category":"Work"}
                    """))
            .andExpect(status().isBadRequest());
    }

    @Test
    void shouldDeleteTask() throws Exception {
        doNothing().when(taskService).deleteTask(1L);

        mockMvc.perform(delete("/api/tasks/1"))
            .andExpect(status().isNoContent());
    }
}
💡 jsonPath 語法 $ = JSON 根物件
$.name = 取 name 欄位
$[0] = 陣列第一個元素
$.length() = 陣列長度
和 JS 的 response.name / response[0] 一樣的邏輯。

Step 3：Repository 測試（@DataJpaTest）

建立 repository/TaskRepositoryTest.java：

package com.example.demo.repository;

import com.example.demo.model.Task;
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.autoconfigure.orm.jpa.DataJpaTest;
import static org.assertj.core.api.Assertions.*;

@DataJpaTest
class TaskRepositoryTest {

    @Autowired
    private TaskRepository taskRepo;

    @Test
    void shouldSaveAndFindTask() {
        var saved = taskRepo.save(new Task("Learn JPA"));

        var found = taskRepo.findById(saved.getId());

        assertThat(found).isPresent();
        assertThat(found.get().getTitle()).isEqualTo("Learn JPA");
    }

    @Test
    void shouldFindByCompletedFalse() {
        taskRepo.save(new Task("Done"));
        var undone = new Task("Still going");
        undone.setCompleted(false);
        taskRepo.save(undone);

        var active = taskRepo.findByCompletedFalse();

        assertThat(active).hasSize(2);  // 兩個都未完成（false 是預設值）
    }

    @Test
    void shouldFindByTitleContaining() {
        taskRepo.save(new Task("Spring Boot"));
        taskRepo.save(new Task("Spring Data"));
        taskRepo.save(new Task("Python"));

        var result = taskRepo.findByTitleContainingIgnoreCaseOrderByCreatedAtDesc("Spring");

        assertThat(result).hasSize(2);
    }

    @Test
    void shouldDeleteTask() {
        var saved = taskRepo.save(new Task("Temp"));
        taskRepo.deleteById(saved.getId());

        assertThat(taskRepo.findById(saved.getId())).isEmpty();
    }
}

Step 4：整合測試（@SpringBootTest）

改寫已存在的 DemoApplicationTests.java：

package com.example.demo;

import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.boot.test.web.client.TestRestTemplate;
import org.springframework.http.*;
import static org.assertj.core.api.Assertions.*;

@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
class DemoApplicationTests {

    @Autowired
    private TestRestTemplate restTemplate;

    @Test
    void contextLoads() {
        // 確認容器能正常啟動（Spring Initializr 預設的測試）
    }

    @Test
    void shouldCreateAndGetTask() {
        // Create
        var request = new java.util.HashMap<>();
        request.put("title", "Integration Test");
        request.put("category", "Test");

        var createResp = restTemplate.postForEntity(
            "/api/tasks", request, String.class
        );
        assertThat(createResp.getStatusCode()).isEqualTo(HttpStatus.CREATED);

        // GetAll
        var getAllResp = restTemplate.getForEntity("/api/tasks", String.class);
        assertThat(getAllResp.getStatusCode()).isEqualTo(HttpStatus.OK);
        assertThat(getAllResp.getBody()).contains("Integration Test");
    }
}

Step 5：執行所有測試

在 IntelliJ 中：

  • 執行單一測試：點方法左側的綠色 ▶
  • 執行整個測試類別：點 class 名稱左側的 ▶
  • 執行全部測試：在專案根目錄按 Ctrl+Shift+F10，或終端機執行：
# 在專案根目錄（build.gradle 所在處）
./gradlew test

全部綠色 ✓ 的話，你的測試覆蓋了三個層級。

課後檢查清單

  • TaskServiceTest 用 Mockito 測了查詢、更新、刪除（不啟動 Spring）
  • TaskControllerTest 用 @WebMvcTest + MockMvc 測了 200/404/201/400
  • TaskRepositoryTest 用 @DataJpaTest 測了衍生查詢方法
  • DemoApplicationTests 用 @SpringBootTest + TestRestTemplate 做整合測試
  • 全部測試綠色（或在 Gradle 中看到 BUILD SUCCESSFUL）

📌 四層測試速查

Mockito 單元@ExtendWith + @Mock + @InjectMocks
測 Service 邏輯，mock Repository
@WebMvcTestMockMvc + @MockitoBean
測 Controller HTTP 層，mock Service
@DataJpaTest自動 H2 + rollback
測 Repository 查詢，驗證衍生方法
@SpringBootTestTestRestTemplate + RANDOM_PORT
測完整串接，少而精
