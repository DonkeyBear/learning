Java OOP 核心：class、interface、繼承、record

你寫過 TypeScript 的 class 和 interface，也用過 Python 的繼承。Java 的 OOP 機制比 TS 更嚴格、更豐富——這堂課聚焦在和 TS/Python 不同的地方，以及 Spring Boot 開發中最常見的 OOP 模式。

1. 類別（Class）：比 TS 多一個「套件層級」

Java class 的基本結構你已經在 T01 看過。這裡聚焦在 存取修飾子（access modifiers）——這是 Java 和 TS/Python 差異最大的地方。

修飾子JavaTypeScript說明
public ✅ ✅ 到處都可以存取
private ✅ ✅ 只有同一個 class 內可以存取
protected ✅ ✅ 同 class + 子類別 + 同套件（Java 多了這個）
（無修飾子） package-private ❌ 不存在 只有同一個套件（package）內可以存取
⚠️ TS 開發者注意 Java 的 預設（不加任何修飾子）= package-private，不是 public！在 TS 中不寫修飾子預設是 public，這點和 Java 完全不同。Spring Boot 專案中你會大量使用 package-private 來控制模組內的可見性。

2. 繼承（Inheritance）：單一繼承、多介面實作

Java 的繼承規則比 TS/Python 嚴格：

// ✅ 一個 class 只能繼承一個父類別
public class Dog extends Animal { }

// ✅ 但可以實作多個介面
public class Dog extends Animal implements Runnable, Eatable { }

// ❌ Java 不支援多重繼承（C++ 有，Python 有，Java 沒有）
// public class Dog extends Animal, Pet { }  ← 編譯錯誤！

對比：

  • Python 支援多重繼承（class Dog(Animal, Pet)）——Java 沒有
  • TS 也是單一繼承，所以這點 TS 開發者應該很熟悉
  • Java 用 interface 多重實作來彌補單一繼承的限制

方法覆寫與 @Override

public class Animal {
    public String makeSound() {
        return "Some sound";
    }
}

public class Dog extends Animal {
    @Override          // ← 這個 annotation 是強烈建議的
    public String makeSound() {
        return "Woof!";
    }
}

@Override 是一個編譯器檢查標記（annotation）。如果你加上它但實際上沒有覆寫到父類別的方法（例如拼錯字），編譯器會直接報錯。這比 TS 的 override 關鍵字（TS 4.3+）更早就存在。

💡 順帶一提：@ 標記叫 Annotation Java 中 @Override、@Deprecated、@SuppressWarnings 這些以 @ 開頭的東西稱為 Annotation（註解）。它們和 TS 的 decorator（@Component、@Injectable）是對應概念。Spring Boot 大量使用自訂 annotation，例如 @RestController、@Autowired、@Entity——這些我們會在 Phase 2 深入。

3. 抽象類別與 final

abstract — 不完整的類別

// 抽象類別：不能直接 new，必須被繼承
public abstract class Shape {
    // 抽象方法：沒有實作，子類別必須覆寫
    public abstract double area();

    // 抽象類別中也可以有具體方法
    public String description() {
        return "This is a shape with area: " + area();
    }
}

public class Circle extends Shape {
    private double radius;

    public Circle(double r) { this.radius = r; }

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
}

和 TS 的 abstract class 幾乎一樣。Spring Boot 中你不會頻繁寫 abstract class，但很多 Spring 內部的基礎類別是 abstract 的（例如 AbstractSecurityInterceptor）。

final — 禁止修改

final 在 Java 中有三種用法，比 TS 的 readonly 或 const 涵蓋更廣：

// 1. final 變數 → 不可重新賦值（類似 const）
final int MAX = 100;

// 2. final 方法 → 子類別不能覆寫
public final String getSecret() { return "..."; }

// 3. final class → 不能被繼承（類似 TS 中沒有 export 的 class）
public final class StringUtils { }
🎯 面試相關 final 的三種語意是常見面試題。另一個常見問題是：String 為什麼是 final？答案：安全性（不可變的 String 可以在多執行緒間安全共享）和效能（String pool）。這個在 T08 執行緒安全課中會再深入。

4. Interface：Java 的終極抽象

TS 的 interface 和 Java 的 interface 在概念上非常相似——定義一份「合約」，讓 class 來實作。但 Java 的 interface 有一些 TS 沒有的特性：

// 定義一個 interface
public interface PaymentService {
    // 抽象方法（預設就是 public abstract）
    boolean processPayment(double amount);

    // Java 8+：default 方法 —— 可以有預設實作！
    default void logPayment(double amount) {
        System.out.println("Processing: $" + amount);
    }

    // Java 8+：static 方法
    static PaymentService createDefault() {
        return new CreditCardPaymentService();
    }
}

// 實作 interface
public class CreditCardPaymentService implements PaymentService {
    @Override
    public boolean processPayment(double amount) {
        // 實際的信用卡處理邏輯
        return true;
    }
}
💡 TS 開發者注意：default 方法 Java 8 的 default 方法是 TS interface 沒有的概念——它讓你在 interface 中提供預設實作，既有的實作類別不需要修改就能獲得新方法。這是 Java 在向後相容性上的一大創新。Spring Data JPA 的 findById()、save() 等標準方法就是透過 interface default 方法提供的——你不用寫實作，框架幫你做好了。

interface vs abstract class：何時用哪個？

特性interfaceabstract class
多重實作/繼承✅ 可實作多個❌ 只能繼承一個
可以有建構子❌✅
可以有狀態（欄位）❌（只有常數）✅

5. Record：Java 版的 data class

這是 Java 14+（JDK 21 當然支援）最令人興奮的新特性之一。如果你懷念 Python 的 dataclass 或 TS 中一行定義完的簡單型別，你一定會喜歡 record。

沒有 record 之前，Java 要定義一個單純的資料載體（DTO）非常冗長：

// 😩 傳統寫法：光是 name 和 age 兩個欄位就要 40+ 行程式碼
public class UserDto {
    private final String name;
    private final int age;

    public UserDto(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() { return name; }
    public int getAge() { return age; }

    @Override
    public boolean equals(Object o) { /* ... */ }

    @Override
    public int hashCode() { /* ... */ }

    @Override
    public String toString() { /* ... */ }
}

// ✨ record 寫法：一行搞定，編譯器自動產生所有方法
public record UserDto(String name, int age) { }

Record 自動提供：

  • 建構子（canonical constructor）
  • 所有欄位的 getter（方法名直接就是欄位名，沒有 get 前綴！例如 userDto.name()）
  • equals()、hashCode()、toString()
  • 欄位是 final（不可變——immutable）
🎯 面試相關 Record 在 Spring Boot 中最常見的用途是 DTO（Data Transfer Object）——Controller 接收和回傳的請求/回應物件。面試中如果被問到「如何減少 Java 的 boilerplate code」，record 是標準答案之一（另一個是 Lombok）。

Record 對比 TS/Python

JavaTypeScriptPython
public record Point(int x, int y) { } type Point = { x: number; y: number }; @dataclass
class Point:
    x: int

6. 方法多載（Method Overloading）

Java 允許同一個 class 中有多個同名但參數不同的方法。這是 TS/Python 都沒有的特性（Python 只能用預設參數模擬）：

public class Calculator {
    // 同名方法，不同參數型別或數量
    public int add(int a, int b) {
        return a + b;
    }

    public double add(double a, double b) {
        return a + b;
    }

    public int add(int a, int b, int c) {
        return a + b + c;
    }
}

// 使用時編譯器會根據你傳入的參數自動匹配版本
var calc = new Calculator();
calc.add(1, 2);        // → 呼叫 int 版本
calc.add(1.5, 2.3);    // → 呼叫 double 版本
calc.add(1, 2, 3);     // → 呼叫三參數版本

Spring 內部大量使用多載——例如 RestTemplate.getForObject() 有十幾個不同參數組合的多載版本。

自我檢驗 🧠

Q1: Java 中一個 class 可以繼承多個父類別嗎？可以實作多個 interface 嗎？

點擊展開答案

class 單一繼承（只能 extends 一個父類別），但可以 implements 多個 interface。這是 Java 解決多重繼承問題的設計取捨：避免了 C++ 的菱形繼承問題，同時保留了多重行為規範的彈性。Spring Boot 中一個典型的 Service 通常 implements 一個 interface，而那個 interface 可能又 extends 多個上層 interface（如 JPA 的 JpaRepository）。

Q2: final 在 Java 中有哪三種用法？

點擊展開答案

① final 變數：不可重新賦值（類似 const）。② final 方法：子類別不可覆寫。③ final class：不可被繼承（例如 String 就是 final class，保證其不可變性和執行緒安全）。

Q3: Record 和一般 class 有什麼本質差異？Spring Boot 中 record 最常用在什麼場景？

點擊展開答案

Record 是不可變的資料載體——所有欄位都是 final，自動產生 constructor、getter（無 get 前綴）、equals/hashCode/toString。Spring Boot 中常用作 DTO（Data Transfer Object），例如 Controller 的請求/回應物件。和一般 class 的差異：record 不能繼承其他 class（隱含 final），也不能被繼承。

📌 關鍵 Takeaway

  1. package-private（無修飾子）是 Java 獨有的可見度層級，TS 開發者最容易忽略
  2. 單一繼承 + 多介面實作是 Java 的核心設計，和 Python 的多重繼承不同
  3. @Override 是編譯器檢查標記，強烈建議每次都加
  4. interface default 方法是 Java 8+ 的關鍵特性，Spring Data 大量依賴它
  5. Record = Java 版的 data class / DTO，一行取代傳統的 40 行 boilerplate
  6. 方法多載（overloading）是 Java 原生特性，TS/Python 沒有
