分層架構實作：Controller → Service → Repository

P04 的 demo 只有兩層（Controller + Service），而且 Service 直接回傳字串。這堂課會把專案擴展為完整的三層架構，加入真正的資料層（Repository），並練習 T05 學到的 @Qualifier 多重實作。

🎯 這堂課的目標 在 P04 的 demo 專案中建立一個「任務管理」REST API，含完整三層 + 介面/實作分離 + 兩種 Repository 實作（記憶體 vs 檔案），用 @Qualifier 切換。

Step 1：建立 Model

建立 model/Task.java：

package com.example.demo.model;

import java.time.LocalDateTime;
import java.util.UUID;

public class Task {
    private final String id;
    private String title;
    private boolean completed;
    private final LocalDateTime createdAt;

    public Task(String title) {
        this.id = UUID.randomUUID().toString().substring(0, 8);
        this.title = title;
        this.completed = false;
        this.createdAt = LocalDateTime.now();
    }

    // === Getters ===
    public String getId() { return id; }
    public String getTitle() { return title; }
    public boolean isCompleted() { return completed; }
    public LocalDateTime getCreatedAt() { return createdAt; }

    // === Setters（只有可變欄位） ===
    public void setTitle(String title) { this.title = title; }
    public void setCompleted(boolean completed) { this.completed = completed; }
}

建立 model/TaskRequest.java（用 record —— T02 教的）：

package com.example.demo.model;

public record TaskRequest(String title) {
    public TaskRequest {
        if (title == null || title.isBlank()) {
            throw new IllegalArgumentException("標題不能為空");
        }
    }
}

Step 2：建立 Repository 層（介面 + 雙實作）

建立 repository/TaskRepository.java（介面）：

package com.example.demo.repository;

import com.example.demo.model.Task;
import java.util.List;
import java.util.Optional;

public interface TaskRepository {
    List<Task> findAll();
    Optional<Task> findById(String id);
    Task save(Task task);
    void deleteById(String id);
}

建立 repository/InMemoryTaskRepository.java（實作 A——主要使用）：

package com.example.demo.repository;

import com.example.demo.model.Task;
import org.springframework.stereotype.Repository;

import java.util.*;
import java.util.concurrent.ConcurrentHashMap;

@Repository
@Primary  // ← 當有多個 TaskRepository Bean 時，這是預設選擇
public class InMemoryTaskRepository implements TaskRepository {
    private final Map<String, Task> store = new ConcurrentHashMap<>();

    @Override
    public List<Task> findAll() {
        return List.copyOf(store.values());
    }

    @Override
    public Optional<Task> findById(String id) {
        return Optional.ofNullable(store.get(id));
    }

    @Override
    public Task save(Task task) {
        store.put(task.getId(), task);
        return task;
    }

    @Override
    public void deleteById(String id) {
        store.remove(id);
    }
}

建立 repository/FileTaskRepository.java（實作 B——@Qualifier 練習用）：

package com.example.demo.repository;

import com.example.demo.model.Task;
import org.springframework.stereotype.Repository;

import java.util.*;

@Repository("fileRepo")  // ← 給 Bean 命名，供 @Qualifier 使用
public class FileTaskRepository implements TaskRepository {
    // 簡化版：實際上應該讀寫檔案，這裡先當作「另一個實作」的佔位
    private final Map<String, Task> store = new LinkedHashMap<>();

    @Override
    public List<Task> findAll() {
        System.out.println("📁 [FileRepo] 從檔案讀取...");
        return List.copyOf(store.values());
    }

    @Override
    public Optional<Task> findById(String id) {
        return Optional.ofNullable(store.get(id));
    }

    @Override
    public Task save(Task task) {
        System.out.println("📁 [FileRepo] 寫入檔案: " + task.getId());
        store.put(task.getId(), task);
        return task;
    }

    @Override
    public void deleteById(String id) {
        store.remove(id);
    }
}
⚠️ ConcurrentHashMap 說明 為什麼 InMemoryTaskRepository 用 ConcurrentHashMap 而不是普通 HashMap？因為 Spring 的 Service/Repository 預設是 singleton——所有 HTTP 請求共享同一個實例。多個請求同時操作 HashMap 會導致資料不一致甚至死循環。ConcurrentHashMap 是執行緒安全的 Map 實作。這正是 T08 要深入的主題。

Step 3：建立 Service 層（介面 + 實作）

建立 service/TaskService.java（介面）：

package com.example.demo.service;

import com.example.demo.model.Task;
import java.util.List;
import java.util.Optional;

public interface TaskService {
    List<Task> getAllTasks();
    Optional<Task> getTask(String id);
    Task createTask(String title);
    Task updateTask(String id, String title, Boolean completed);
    void deleteTask(String id);
}

建立 service/TaskServiceImpl.java：

package com.example.demo.service;

import com.example.demo.model.Task;
import com.example.demo.repository.TaskRepository;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Optional;

@Service
public class TaskServiceImpl implements TaskService {
    private final TaskRepository taskRepository;

    // 建構子注入——Spring 會注入 @Primary 的實作（InMemoryTaskRepository）
    public TaskServiceImpl(TaskRepository taskRepository) {
        this.taskRepository = taskRepository;
    }

    @Override
    public List<Task> getAllTasks() {
        return taskRepository.findAll();
    }

    @Override
    public Optional<Task> getTask(String id) {
        return taskRepository.findById(id);
    }

    @Override
    public Task createTask(String title) {
        var task = new Task(title);
        return taskRepository.save(task);
    }

    @Override
    public Task updateTask(String id, String title, Boolean completed) {
        var task = taskRepository.findById(id)
            .orElseThrow(() -> new RuntimeException("Task not found: " + id));

        if (title != null) task.setTitle(title);
        if (completed != null) task.setCompleted(completed);

        return taskRepository.save(task);
    }

    @Override
    public void deleteTask(String id) {
        taskRepository.deleteById(id);
    }
}
🎯 設計觀察 Service 不知道也不在乎 TaskRepository 是存在記憶體還是檔案——它只依賴 TaskRepository 介面。這就是 T02 說的「針對介面寫程式」在 Spring Boot 中的威力：換 Repository 實作只需改一個 annotation，Service 一行都不用動。

Step 4：建立 Controller

建立 controller/TaskController.java：

package com.example.demo.controller;

import com.example.demo.model.Task;
import com.example.demo.model.TaskRequest;
import com.example.demo.service.TaskService;
import org.springframework.http.HttpStatus;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.server.ResponseStatusException;

import java.util.List;

@RestController
@RequestMapping("/api/tasks")   // ← 所有端點的前綴
public class TaskController {
    private final TaskService taskService;

    public TaskController(TaskService taskService) {
        this.taskService = taskService;
    }

    // GET /api/tasks
    @GetMapping
    public List<Task> getAll() {
        return taskService.getAllTasks();
    }

    // GET /api/tasks/{id}
    @GetMapping("/{id}")
    public Task getOne(@PathVariable String id) {
        return taskService.getTask(id)
            .orElseThrow(() -> new ResponseStatusException(
                HttpStatus.NOT_FOUND, "找不到任務: " + id
            ));
    }

    // POST /api/tasks
    @PostMapping
    @ResponseStatus(HttpStatus.CREATED)  // 回傳 201 而非預設的 200
    public Task create(@RequestBody TaskRequest request) {
        return taskService.createTask(request.title());
    }

    // PUT /api/tasks/{id}
    @PutMapping("/{id}")
    public Task update(
        @PathVariable String id,
        @RequestBody TaskRequest request
    ) {
        return taskService.updateTask(id, request.title(), null);
    }

    // PATCH /api/tasks/{id}/complete
    @PatchMapping("/{id}/complete")
    public Task toggleComplete(@PathVariable String id) {
        var task = taskService.getTask(id)
            .orElseThrow(() -> new ResponseStatusException(
                HttpStatus.NOT_FOUND, "找不到任務: " + id
            ));
        return taskService.updateTask(id, null, !task.isCompleted());
    }

    // DELETE /api/tasks/{id}
    @DeleteMapping("/{id}")
    @ResponseStatus(HttpStatus.NO_CONTENT)  // 204
    public void delete(@PathVariable String id) {
        taskService.deleteTask(id);
    }
}
💡 新的 Annotation 說明
@RequestMapping("/api/tasks")所有方法路徑的前綴
@PathVariable從 URL 路徑取出變數（等於 FastAPI 的 {id}）
@PutMapping完整更新資源
@PatchMapping部分更新（這裡只用來切換完成狀態）
@DeleteMapping刪除資源
@ResponseStatus覆寫預設的 HTTP 狀態碼

Bonus：練習 @Qualifier 切換 Repository

建立一個新的 Controller 來測試 FileTaskRepository：

package com.example.demo.controller;

import com.example.demo.model.Task;
import com.example.demo.repository.TaskRepository;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/file-tasks")
public class FileTaskController {

    private final TaskRepository taskRepository;

    // 指名使用 fileRepo，而非 @Primary 的 InMemoryTaskRepository
    public FileTaskController(
        @Qualifier("fileRepo") TaskRepository taskRepository
    ) {
        this.taskRepository = taskRepository;
    }

    @GetMapping
    public List<Task> getAll() {
        return taskRepository.findAll();
    }

    @PostMapping
    public Task create(@RequestBody com.example.demo.model.TaskRequest req) {
        var task = new Task(req.title());
        return taskRepository.save(task);
    }
}

現在你有兩個 Controller，使用不同的 Repository 實作：

  • /api/tasks → 使用 InMemoryTaskRepository（@Primary）
  • /api/file-tasks → 使用 FileTaskRepository（@Qualifier）

課後檢查清單

  • Controller / Service / Repository 三層各自獨立，依賴方向正確
  • Service 和 Repository 都有對應的 interface + 實作
  • Controller 只透過 Service 介面互動，不直接碰 Repository
  • GET/POST/PUT/PATCH/DELETE 五條 API 都能正常運作
  • @Primary 的 InMemoryTaskRepository 是預設實作
  • @Qualifier("fileRepo") 能切換到另一個實作
  • 了解 @RequestMapping、@PathVariable、@ResponseStatus 用法

📌 標準三層架構速查

Controller@RestController + @RequestMapping
職責：HTTP 請求/回應，參數驗證，呼叫 Service
不寫業務邏輯
Serviceinterface + @Service 實作
職責：業務邏輯、交易管理、協調多個 Repository
不知道 HTTP 的存在
Repositoryinterface + @Repository 實作
職責：資料存取、查詢、持久化
不知道業務邏輯的存在
