Java Collections 與 Stream API

在 JS 中你用 map/filter/reduce 處理陣列，在 Python 中用 list comprehension。Java 的答案是 Stream API——它不僅是語法糖，更是一套完整的函數式資料處理管線。這堂課先補齊 Collections 框架的知識，再進入 Stream API 的核心。

🎯 為什麼面試必考 Stream API 是 Java 8 最重要的變革，也是後端日常開發中處理集合資料的主要手段。面試中常見的考法：「給一個 List<Employee>，找出薪水最高的前 5 個部門」——用 Stream 一行就能解決，用傳統 for 迴圈要寫十幾行。能熟練使用 Stream 是 Java 開發者的基本功。

2. Stream API：Java 的 map/filter/reduce

Stream 不是資料結構——它是一條管線（pipeline），資料從來源流過一連串操作，在終點匯聚成結果。

核心概念：中間操作 vs 終端操作

🔹 中間操作（Intermediate）

回傳一個新的 Stream，懶加載（lazy）——在終端操作觸發前不執行

  • .map()
  • .filter()
  • .sorted()
  • .distinct()
  • .limit() / .skip()

🔸 終端操作（Terminal）

觸發管線執行，結束 Stream（同一個 Stream 只能消費一次）

  • .collect(Collectors.toList())
  • .forEach()
  • .reduce()
  • .count()
  • .findFirst() / .anyMatch()

3. 三語對照：map / filter / reduce

假設有一個數字列表，你要過濾偶數 → 平方 → 加總：

Java StreamJavaScriptPython
List<Integer> nums = List.of(1,2,3,4,5,6); int sum = nums.stream() .filter(n -> n % 2 == 0) .map(n -> n * n) .reduce(0, Integer::sum); const nums = [1,2,3,4,5,6]; const sum = nums .filter(n => n % 2 === 0) .map(n => n * n) .reduce((a,b) => a+b, 0); nums = [1,2,3,4,5,6] squares = [n*n for n in nums if n % 2 == 0] total = sum(squares)

語法比較：

  • JS arrow function n => n * 2 → Java lambda n -> n * 2（箭頭方向不同！）
  • JS filter(n => condition) → Java filter(n -> condition)（完全對應）
  • JS reduce((a,b) => a+b, 0) → Java reduce(0, (a,b) -> a+b)（初始值位置不同）
  • Integer::sum 是方法參考（method reference）—— (a,b) -> a+b 的簡寫

4. Stream 常用操作詳解

filter — 過濾

var names = List.of("Alice", "Bob", "Charlie", "Anna");
var aNames = names.stream()
    .filter(name -> name.startsWith("A"))  // [Alice, Anna]
    .toList();

map — 轉換型別/值

// 取出員工姓名 → 轉大寫
record Employee(String name, int salary) { }

var employees = List.of(
    new Employee("Alice", 70000),
    new Employee("Bob", 50000)
);

var upperNames = employees.stream()
    .map(Employee::name)              // 方法參考：等於 e -> e.name()
    .map(String::toUpperCase)         // 等於 s -> s.toUpperCase()
    .toList();                        // [ALICE, BOB]

sorted — 排序

// 按薪水由高到低排序
var sorted = employees.stream()
    .sorted((a, b) -> Integer.compare(b.salary(), a.salary()))
    .toList();

// 或用 Comparator.comparing（更常見）
var sorted2 = employees.stream()
    .sorted(Comparator.comparing(Employee::salary).reversed())
    .toList();

collect — 匯集到不同容器

import java.util.stream.Collectors;

// 收集成 List
List<String> names = employees.stream()
    .map(Employee::name)
    .collect(Collectors.toList());

// 收集成 Map（name → salary）
Map<String, Integer> nameToSalary = employees.stream()
    .collect(Collectors.toMap(
        Employee::name,
        Employee::salary
    ));

// 分組（groupingBy）
Map<Character, List<Employee>> byFirstLetter = employees.stream()
    .collect(Collectors.groupingBy(
        e -> e.name().charAt(0)
    ));
🔥 面試重點：groupingBy Collectors.groupingBy() 是 SQL 的 GROUP BY 在 Java 中的對應，面試極高頻。例如「把員工按部門分組，計算各部門的平均薪資」就會用到它搭配 Collectors.averagingInt()。

flatMap — 攤平巢狀結構

// 每個員工有多個技能，攤平成單一技能列表
record Employee(String name, List<String> skills) { }

var employees = List.of(
    new Employee("Alice", List.of("Java", "SQL")),
    new Employee("Bob",   List.of("Python", "Docker"))
);

var allSkills = employees.stream()
    .flatMap(e -> e.skills().stream())  // 把多個 List 攤平成一個 Stream
    .distinct()                          // 去重
    .toList();                           // [Java, SQL, Python, Docker]

6. 原始型別 Stream（效能優化）

因為 List<int> 在 Java 中不合法（泛型不能放 primitive），Java 提供了專用的原始型別 Stream，避免自動裝箱（autoboxing）的效能損耗：

// ❌ 一般 Stream：每個 int 都要裝箱成 Integer，浪費效能
int sum = List.of(1, 2, 3).stream().reduce(0, Integer::sum);

// ✅ IntStream：直接操作原始 int，零裝箱
int sum2 = IntStream.of(1, 2, 3).sum();
int sum3 = IntStream.range(1, 101).sum();          // 1 加到 100
int sum4 = IntStream.rangeClosed(1, 100).sum();    // 1 加到 100（含 100）

同理還有 LongStream 和 DoubleStream。Spring Boot 中處理大量資料（例如報表計算）時會用到。

自我檢驗 🧠

Q1: Stream 的中間操作和終端操作有何本質差異？為什麼 Stream 只能消費一次？

點擊展開答案

中間操作（map/filter/sorted）回傳新的 Stream，是懶加載的——在終端操作觸發前不會執行。終端操作（collect/reduce/forEach）觸發整個管線執行並結束 Stream。Stream 只能消費一次，因為它背後是對資料來源的迭代器（Iterator），用完就消耗完了。如果需要重複使用，必須從原始 Collection 重新建立 Stream。

Q2: 用 Stream 將 List<String> 過濾掉空字串、轉大寫、收集到新的 List。寫出程式碼。

點擊展開答案 
var result = list.stream()
    .filter(s -> !s.isBlank())
    .map(String::toUpperCase)
    .toList();  // Java 16+，等於 collect(Collectors.toList())

Q3: flatMap 和 map 的差別是什麼？什麼時候用 flatMap？

點擊展開答案

map：一對一轉換（T → R），每個元素轉成一個新元素。flatMap：一對多攤平（T → Stream<R>），每個元素轉成一個 Stream，再將所有 Stream 攤平成單一 Stream。當你的映射結果本身就是集合（例如：員工有多個技能、訂單有多個商品），想攤平成單一列表時用 flatMap。JS 的 Array.flatMap() 是一樣的概念。

📌 關鍵 Takeaway

  1. List / Set / Map 三本柱，Spring Boot 中輪流登場：List 查詢結果、Set 去重 ID、Map 快取/分組
  2. Stream = 管線：資料來源 → 中間操作（lazy）→ 終端操作（觸發執行）
  3. JS 的 map/filter/reduce 思考方式可直接套用到 Stream，注意箭頭方向（-> vs =>）和 Integer::sum 方法參考
  4. groupingBy 是 SQL GROUP BY 的對應，面試萬年考題
  5. Stream 只能消費一次——用完必須從原始 Collection 重建
  6. .toList()（Java 16+）是 .collect(Collectors.toList()) 的簡寫
