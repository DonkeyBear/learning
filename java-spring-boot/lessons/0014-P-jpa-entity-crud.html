JPA Entity 設計 + Repository CRUD 實作

這堂課會把 P05 的記憶體資料層換成真正的資料庫。你會親眼看到——只定義一個 interface，Spring 就自動幫你生出完整的 save/findAll/findById/delete。

🎯 這堂課的目標 整合 H2 記憶體資料庫 → 把 Task 改成真正的 @Entity → 定義一對多關聯（Category ↔ Task）→ 用 JpaRepository 取代手寫 Repository → 透過 H2 Console 親眼看到資料表。

Step 1：加入 JPA 依賴與設定

build.gradle 的 dependencies 區塊加入三行：

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    implementation 'org.springframework.boot:spring-boot-starter-data-jpa'  // JPA
    runtimeOnly 'com.h2database:h2'                    // H2 資料庫
    // 不需要 spring-boot-starter-test（本來就有了）
}

在 application.properties（src/main/resources/）加入：

# H2 記憶體資料庫
spring.datasource.url=jdbc:h2:mem:taskdb
spring.datasource.driverClassName=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=

# H2 Web Console（讓你用瀏覽器查看資料表）
spring.h2.console.enabled=true
spring.h2.console.path=/h2-console

# JPA 自動建表（開發階段用，正式環境應關閉）
spring.jpa.hibernate.ddl-auto=create-drop

# 在 Console 顯示 SQL（開發除錯用）
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
💡 這些設定的意義
  • H2：Java 寫的嵌入式資料庫，存在記憶體中，重啟就消失——適合開發和學習，不需要安裝 MySQL
  • ddl-auto=create-drop：啟動時根據 @Entity 自動建立資料表，關閉時刪除。開發用 create-drop，正式環境用 validate 或 none
  • show-sql=true：在 Console 印出 Hibernate 產生的 SQL，學習 JPA 的絕佳工具

Reload Gradle（IntelliJ 右下角會彈出提示，或點 Gradle 面板的刷新圖示 🔄）。

Step 2：改造 Task 為 JPA Entity

把 model/Task.java 改寫為 JPA Entity：

package com.example.demo.model;

import jakarta.persistence.*;
import java.time.LocalDateTime;

@Entity
@Table(name = "tasks")
public class Task {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;                        // ← 改用 Long，資料庫自動遞增

    @Column(nullable = false, length = 200)
    private String title;

    @Column(nullable = false)
    private boolean completed = false;

    @Column(name = "created_at", nullable = false)
    private LocalDateTime createdAt;

    @ManyToOne(fetch = FetchType.LAZY)      // ← 關聯到 Category
    @JoinColumn(name = "category_id")
    private Category category;

    // === JPA 要求的無參建構子 ===
    protected Task() { }

    // === 你用的建構子 ===
    public Task(String title) {
        this.title = title;
        this.createdAt = LocalDateTime.now();
    }

    // === Getters & Setters（JPA 需要 setter 來設定欄位值） ===
    public Long getId() { return id; }
    public String getTitle() { return title; }
    public void setTitle(String title) { this.title = title; }
    public boolean isCompleted() { return completed; }
    public void setCompleted(boolean completed) { this.completed = completed; }
    public LocalDateTime getCreatedAt() { return createdAt; }
    public Category getCategory() { return category; }
    public void setCategory(Category category) { this.category = category; }
}

Step 3：建立 Category Entity（一對多）

建立 model/Category.java：

package com.example.demo.model;

import jakarta.persistence.*;
import java.util.ArrayList;
import java.util.List;

@Entity
@Table(name = "categories")
public class Category {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, unique = true, length = 50)
    private String name;

    // 一對多：一個 Category 有多個 Task
    // mappedBy = "category" → 指向 Task.java 中的 category 欄位
    // cascade = ALL → 刪除 Category 時連帶刪除其下所有 Task
    @OneToMany(mappedBy = "category", cascade = CascadeType.ALL, orphanRemoval = true)
    private List<Task> tasks = new ArrayList<>();

    protected Category() { }

    public Category(String name) {
        this.name = name;
    }

    // === Getters ===
    public Long getId() { return id; }
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public List<Task> getTasks() { return tasks; }

    // === 雙向關聯 helper ===
    public void addTask(Task task) {
        tasks.add(task);
        task.setCategory(this);
    }

    public void removeTask(Task task) {
        tasks.remove(task);
        task.setCategory(null);
    }
}
🔍 雙向關聯的 addTask/removeTask helper 當你設定 task.setCategory(category) 時，JPA 會更新外鍵，但 category.getTasks() 不會自動新增。addTask() helper 確保兩邊同步——這不是 JPA 強制的，但能避免許多難以追蹤的 bug。

Step 4：用 JpaRepository 取代手寫 Repository

刪掉舊的 InMemoryTaskRepository.java 和 FileTaskRepository.java。建立新的 JPA Repository：

repository/TaskRepository.java：

package com.example.demo.repository;

import com.example.demo.model.Task;
import org.springframework.data.jpa.repository.JpaRepository;
import java.util.List;

public interface TaskRepository extends JpaRepository<Task, Long> {
    // JpaRepository 已自動提供：findAll, findById, save, deleteById, count...

    // 查詢未完成的任務
    List<Task> findByCompletedFalse();

    // 依分類 ID 查詢
    List<Task> findByCategoryId(Long categoryId);

    // 搜尋標題（LIKE %keyword%），依建立時間降冪
    List<Task> findByTitleContainingIgnoreCaseOrderByCreatedAtDesc(String keyword);
}

repository/CategoryRepository.java：

package com.example.demo.repository;

import com.example.demo.model.Category;
import org.springframework.data.jpa.repository.JpaRepository;
import java.util.Optional;

public interface CategoryRepository extends JpaRepository<Category, Long> {
    Optional<Category> findByName(String name);
}

注意：你沒有建立任何實作類別。只有兩個 interface。Spring Data JPA 會在執行時自動產生實作。

Step 5：改寫 Service 層

修改 TaskServiceImpl.java，改用真實的 JPA Repository：

package com.example.demo.service;

import com.example.demo.model.Category;
import com.example.demo.model.Task;
import com.example.demo.repository.CategoryRepository;
import com.example.demo.repository.TaskRepository;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;
import java.util.Optional;

@Service
@Transactional(readOnly = true)  // ← 預設 readOnly，寫入方法單獨標記
public class TaskServiceImpl implements TaskService {

    private final TaskRepository taskRepo;
    private final CategoryRepository categoryRepo;

    public TaskServiceImpl(TaskRepository taskRepo, CategoryRepository categoryRepo) {
        this.taskRepo = taskRepo;
        this.categoryRepo = categoryRepo;
    }

    @Override
    public List<Task> getAllTasks() {
        return taskRepo.findAll();
    }

    @Override
    public Optional<Task> getTask(Long id) {   // ← 改成 Long
        return taskRepo.findById(id);
    }

    @Override
    @Transactional  // ← 標記為寫入交易
    public Task createTask(String title, String categoryName) {
        var category = categoryRepo.findByName(categoryName)
            .orElseGet(() -> categoryRepo.save(new Category(categoryName)));

        var task = new Task(title);
        category.addTask(task);        // 雙向關聯 helper
        return taskRepo.save(task);    // ← JPA 的 save()
    }

    @Override
    @Transactional
    public Task updateTask(Long id, String title, Boolean completed) {
        var task = taskRepo.findById(id)
            .orElseThrow(() -> new RuntimeException("Task not found: " + id));
        if (title != null) task.setTitle(title);
        if (completed != null) task.setCompleted(completed);
        return taskRepo.save(task);
    }

    @Override
    @Transactional
    public void deleteTask(Long id) {
        taskRepo.deleteById(id);
    }

    // 新增：查詢特定分類的任務
    public List<Task> getTasksByCategory(Long categoryId) {
        return taskRepo.findByCategoryId(categoryId);
    }

    // 新增：搜尋任務
    public List<Task> searchTasks(String keyword) {
        return taskRepo.findByTitleContainingIgnoreCaseOrderByCreatedAtDesc(keyword);
    }
}

同步更新 TaskService interface 的方法簽章（String id → Long id）。

Step 6：更新 Controller

修改 TaskController.java，改為 Long id、支援分類：

// TaskRequest 更新為可選分類
public record TaskRequest(String title, String category) {
    public TaskRequest {
        if (title == null || title.isBlank()) {
            throw new IllegalArgumentException("標題不能為空");
        }
    }
}

// POST /api/tasks
@PostMapping
@ResponseStatus(HttpStatus.CREATED)
public Task create(@RequestBody TaskRequest request) {
    String cat = request.category() != null ? request.category() : "預設";
    return taskService.createTask(request.title(), cat);
}

// GET /api/tasks/search?q=關鍵字
@GetMapping("/search")
public List<Task> search(@RequestParam String q) {
    return taskService.searchTasks(q);
}

// GET /api/tasks?completed=false
@GetMapping
public List<Task> getAll(@RequestParam(defaultValue = "all") String filter) {
    // 依 filter 參數決定回傳全部或未完成
    return switch (filter) {
        case "active" -> taskService.getActiveTasks();
        default -> taskService.getAllTasks();
    };
}

🤔 JpaRepository 報錯？

常見原因：① Entity 缺無參建構子 ② @Id 型別和 JpaRepository<T, ID> 的 ID 不一致 ③ 衍生查詢方法名稱拼錯（欄位名必須和 Entity 屬性完全一致）。把錯誤訊息貼上來。
