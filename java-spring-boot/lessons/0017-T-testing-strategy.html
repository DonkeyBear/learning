Spring Boot 測試策略：JUnit、Mock、MockMvc

你寫過前端的測試（Jest、Vitest、React Testing Library）。Spring Boot 的測試生態有對應的工具，但分層更細——因為後端的三層架構需要不同層級的測試策略。

🎯 學完這堂課你要能回答 「Spring Boot 有哪些測試層級？」「@WebMvcTest 和 @SpringBootTest 的差異？」「何時用 Mock、何時用真實資料庫？」「MockMvc 是做什麼的？」

1. 測試金字塔（Spring Boot 版）

        ┌──────┐
        │ E2E  │  @SpringBootTest + TestRestTemplate（少）
        ├──────┤
       ┌┤ 整合 │┐ @SpringBootTest / @DataJpaTest（中）
       │├──────┤│
      ┌┤│ Web  │├┐ @WebMvcTest + MockMvc（多）
      │├──────┤│
     ┌┤││ 單元│├│┐ JUnit 5 + Mockito（最多）
     └┴┴──────┴┴┘
層級Annotation啟動什麼速度JS 對應
單元測試（無，純 JUnit）不啟動 Spring⚡ 毫秒Jest 單元測試
Web 層@WebMvcTest只啟動 Controller 層⚡ 幾百毫秒Supertest
資料層@DataJpaTest只啟動 JPA 層⚡ 幾百毫秒無直接對應
整合測試@SpringBootTest啟動整個容器🐢 數秒E2E (Playwright)

2. Service 層單元測試（JUnit 5 + Mockito）

最快速、最基礎的測試——不啟動 Spring 容器，用 Mockito 模擬依賴：

import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.extension.ExtendWith;
import org.mockito.InjectMocks;
import org.mockito.Mock;
import org.mockito.junit.jupiter.MockitoExtension;

@ExtendWith(MockitoExtension.class)   // ← 啟用 Mockito
class UserServiceTest {

    @Mock                              // ← 自動建立 mock 物件
    private UserRepository userRepo;

    @InjectMocks                       // ← 自動將 mocks 注入到被測對象
    private UserService userService;

    @Test
    void shouldReturnUserByName() {
        // Arrange
        var user = new User("Alice", "alice@example.com");
        when(userRepo.findByEmail("alice@example.com"))
            .thenReturn(Optional.of(user));

        // Act
        var result = userService.findByEmail("alice@example.com");

        // Assert
        assertThat(result).isPresent();
        assertThat(result.get().getName()).isEqualTo("Alice");
        verify(userRepo).findByEmail("alice@example.com");  // 驗證有被呼叫
    }

    @Test
    void shouldThrowWhenUserNotFound() {
        when(userRepo.findByEmail("nobody@example.com"))
            .thenReturn(Optional.empty());

        assertThrows(UserNotFoundException.class, () ->
            userService.findByEmail("nobody@example.com")
        );
    }
}
💡 Jest 對照
@Mock↔ jest.fn()
when(...).thenReturn(...)↔ mock.mockReturnValue(...)
verify()↔ expect(mock).toHaveBeenCalled()
@InjectMocks↔ 無——Mockito 自動根據型別注入

3. Controller 層測試（@WebMvcTest + MockMvc）

只啟動 Controller 層，不啟動 Service 和資料庫——快速驗證 HTTP 層的行為：

import org.springframework.test.web.servlet.MockMvc;

@WebMvcTest(UserController.class)  // ← 只載入這個 Controller
class UserControllerTest {

    @Autowired
    private MockMvc mockMvc;           // ← 模擬 HTTP 請求

    @MockitoBean                        // ← Mock 掉 Service（Spring Boot 3.4+）
    private UserService userService;    //    舊版用 @MockBean

    @Test
    void shouldReturnUser() throws Exception {
        var user = new User("Alice", "alice@example.com");
        when(userService.getUser(1L)).thenReturn(Optional.of(user));

        mockMvc.perform(get("/api/users/1"))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.name").value("Alice"))
            .andExpect(jsonPath("$.email").value("alice@example.com"));
    }

    @Test
    void shouldCreateUser() throws Exception {
        var request = new UserRequest("Bob", "bob@example.com");
        when(userService.createUser(any())).thenReturn(new User("Bob", "bob@example.com"));

        mockMvc.perform(post("/api/users")
                .contentType(MediaType.APPLICATION_JSON)
                .content("""
                    {"name":"Bob","email":"bob@example.com"}
                """))
            .andExpect(status().isCreated())
            .andExpect(jsonPath("$.name").value("Bob"));
    }

    @Test
    void shouldReturn404WhenUserNotFound() throws Exception {
        when(userService.getUser(999L)).thenReturn(Optional.empty());

        mockMvc.perform(get("/api/users/999"))
            .andExpect(status().isNotFound());
    }
}
🔥 面試關鍵：@WebMvcTest 載入了什麼？ @WebMvcTest 只載入 Controller 和 MVC 基礎設施（Jackson、Validator、ExceptionHandler 等）。不載入 @Service、@Repository、完整的 ApplicationContext。這表示 Controller 測試中你必須 mock 掉所有 Service 依賴。好處是啟動極快（< 1 秒），適合 TDD 循環。

4. Repository 層測試（@DataJpaTest）

只啟動 JPA 層，用嵌入資料庫驗證查詢：

@DataJpaTest   // ← 只載入 JPA Entity、Repository；預設使用 H2
class UserRepositoryTest {

    @Autowired
    private UserRepository userRepo;

    @Test
    void shouldFindByEmail() {
        userRepo.save(new User("Alice", "alice@example.com"));

        var result = userRepo.findByEmail("alice@example.com");

        assertThat(result).isPresent();
        assertThat(result.get().getName()).isEqualTo("Alice");
    }

    @Test
    void shouldExecuteDerivedQuery() {
        userRepo.save(new User("Alice", "alice@example.com"));
        userRepo.save(new User("Bob",   "bob@example.com"));
        userRepo.save(new User("Alex",  "alex@example.com"));

        var result = userRepo.findByNameContainingOrderByNameAsc("Al");

        assertThat(result).hasSize(2);
        assertThat(result.get(0).getName()).isEqualTo("Alex");  // 排序驗證
    }
}
💡 @DataJpaTest 的細節
  • 預設使用 H2 記憶體資料庫（即使你真的用 MySQL）——適合測試衍生查詢的語意是否正確
  • 每個 @Test 預設 rollback——測試之間不互相汙染
  • 如果需要用真實資料庫測試（例如 MySQL 特有的 SQL 語法），加 @AutoConfigureTestDatabase(replace = NONE)

5. 整合測試（@SpringBootTest）

啟動完整的 Spring 容器，適合測試跨層的真實流程：

@SpringBootTest(webEnvironment = WebEnvironment.RANDOM_PORT)
class UserApiIntegrationTest {

    @Autowired
    private TestRestTemplate restTemplate;  // ← 真實 HTTP client

    @Test
    void shouldCreateAndRetrieveUser() {
        // Create
        var request = new UserRequest("Alice", "alice@example.com");
        var createResp = restTemplate.postForEntity("/api/users", request, User.class);
        assertThat(createResp.getStatusCode()).isEqualTo(HttpStatus.CREATED);

        // Retrieve
        var getResp = restTemplate.getForEntity("/api/users/{id}", User.class, 1L);
        assertThat(getResp.getBody().getName()).isEqualTo("Alice");
    }
}
⚠️ @SpringBootTest 很慢 每次啟動完整容器需要數秒。在大型專案中整合測試只佔測試總數的 10-20%。面試中說「整合測試貴但必要」「用 Testcontainers 取代 H2 做真實資料庫測試」會加分。

6. 何時用哪種測試？

你要測的是...用...因為...
複雜的業務邏輯JUnit + Mockito最快、最純粹、不依賴框架
Controller 的參數驗證/HTTP 狀態碼@WebMvcTest + MockMvc只測 HTTP 層，mock 掉 Service
衍生查詢方法是否正確@DataJpaTest只測 JPA，用 H2 快速驗證
整個流程：Controller→Service→DB@SpringBootTest真實的完整流程
交易 rollback、並行鎖行為@SpringBootTest需要完整容器才能測交易

自我檢驗 🧠

Q1: @WebMvcTest 和 @SpringBootTest 的核心差異是什麼？

點擊展開答案

@WebMvcTest 只載入 Controller 和 MVC 基礎設施（不載入 Service、Repository），用 MockMvc 模擬 HTTP 請求，啟動 < 1 秒。@SpringBootTest 載入完整的 ApplicationContext（所有 Bean、資料庫連線等），適合真實流程測試，但啟動需數秒。前者測「Controller 收到請求後的行為」，後者測「整個系統的真實行為」。

Q2: Service 層的單元測試為什麼不建議啟動 Spring 容器？

點擊展開答案

Service 層的單元測試只測業務邏輯，不需要 HTTP 或資料庫。用 Mockito mock 掉依賴（Repository），測試可以在毫秒內完成，且完全獨立於框架。如果每個 Service 測試都啟動 Spring 容器，一個專案的測試可能要跑好幾分鐘——這會讓開發者不願意跑測試。TDD 的基本要求就是測試必須快到可以隨手執行。

Q3: 什麼時候該用 @SpringBootTest 而非 @WebMvcTest？

點擊展開答案

① 需要測試 Controller → Service → Repository 的真實串接 ② 需要測試 @Transactional 的交易行為（rollback、隔離級別）③ 需要測試並行鎖（@Version、@Lock）④ 需要測試自訂的異常處理和過濾器鏈。簡單說：當你需要驗證「整個系統組裝起來後的行為」時，才用 @SpringBootTest。

📌 關鍵 Takeaway

  1. 單元測試（Mockito）→ Service 層：最快，mock 所有依賴
  2. @WebMvcTest → Controller 層：只載入 MVC，mock Service
  3. @DataJpaTest → Repository 層：只載入 JPA，用 H2 驗證查詢
  4. @SpringBootTest → 整合測試：完整容器，測真實串接，少而精
  5. MockMvc 是 Spring MVC 的測試工具，不需要啟動真實 HTTP server
  6. TestRestTemplate 是 @SpringBootTest 中的真實 HTTP client
