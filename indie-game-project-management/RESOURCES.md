# 遊戲開發專案管理 Resources

## Knowledge

- [The Solo Dev QA Stack — GamineAI](https://gamineai.com/blog/the-solo-dev-qa-stack-a-reusable-bug-triage-and-release-notes-workflow-that-scales)
  單人開發者的 bug 分類（P0-P3）、發布筆記撰寫、Steam 上架前檢查清單。使用時機：接近發行階段、需要建立 QA 流程時。

- [Which tool(s) do you use to manage your projects? — Godot Forum](https://forum.godotengine.org/t/which-tool-s-do-you-use-to-manage-your-projects/112037/)
  Godot 開發者社群實際使用的專案管理工具討論。使用時機：選擇或更換管理工具時參考真實使用者的意見。

- [Project Management for Game Development — University of Calgary (PDF)](https://cspages.ucalgary.ca/~bdstephe/585_W11/d504_project_management.pdf)
  遊戲開發專案管理的學術課程講義，涵蓋 feature backlog、估算方法、依賴管理、buffer 規劃。使用時機：需要系統性理解遊戲開發管理基礎時。

- [Codecks — Game Dev Project Management Tool](https://www.codecks.io/)
  專為遊戲開發設計的專案管理工具，以卡牌遊戲隱喻組織 tasks 和 milestones。使用時機：需要一個比通用工具更貼近遊戲開發流程的選項。

- [SoloMode — All-in-one tool for solo game devs](https://www.producthunt.com/products/solomode)
  專為單人開發者打造的 all-in-one 工具：Kanban + GDD 編輯器 + Steam 市場研究 + milestone 追蹤。使用時機：需要一站式解決方案，不想在多個工具之間切換。

## Wisdom (Communities)

- [r/gamedev](https://reddit.com/r/gamedev)
  最大的遊戲開發社群，涵蓋 indie 到 AAA。使用時機：專案管理、發行策略、Steam 上架經驗等一般性問題。

- [Godot Forum — Project Management category](https://forum.godotengine.org/)
  Godot 官方論壇，有許多 Godot 開發者分享他們的 workflow 和管理經驗。使用時機：Godot 特有的管理問題、尋找同樣用 Godot 的開發者。

- [Indie Game Developer — Discord communities]
  多個 indie game dev Discord 伺服器（如 Game Dev League、Indie Game Academy）提供即時交流。使用時機：需要快速回饋、尋找 accountability partner、討論具體的關卡設計管理問題。

## Gaps

- 尚未找到針對「點擊式冒險／密室逃脫」遊戲類型的專案管理資源（這類遊戲的內容生產 pipeline 與動作遊戲或 RPG 差異很大）
- 中文資源稀缺——大部分高品質的遊戲開發管理資源為英文
- 缺乏針對 Godot 引擎特有的內容 pipeline 管理指南
