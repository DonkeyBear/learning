雙板系統

📋 願景板 Vision Board
視角玩家看到的最終體驗
粒度Feature 層級
更新每個工作週期結束後
範例「玩家能組合道具解謎」
⚙️ 執行板 Execution Board
視角開發者具體要執行的工作
粒度2 小時 – 3 天的任務
更新每天
範例「繪製書架三種狀態的素材」

任務粒度判斷

訊號處理
太大 無法估算、不知從何開始、拖好幾天還在「進行中」 依五維度繼續拆解
剛好 2h–3d 可完成、描述清楚、只有一個負責人 ✅ 放進執行板
太小 不到 2 小時、管理 overhead 不值得 合併到相關任務

密室逃脫 · 任務拆解五維度

維度自問產出物
🔍 謎題鏈解謎邏輯？觸發順序？提示在哪？腳本、狀態機、提示文字
🎨 場景素材需要哪些視覺元素？幾種狀態？背景、道具 sprite、UI 元件
💻 互動腳本玩家做 X → 發生 Y？點擊回應、道具組合、過場動畫
🔊 音效氛圍哪些動作需要音效回饋？BGM、SFX、環境音
🧪 測試如何確認真的可以玩？通關測試記錄、卡關點清單

常見反模式

反模式危險修正
「做 Level 1」太模糊，不可估算、不可追蹤拆成多個謎題 → 每個謎題再拆五維度
同時看兩塊板PM 腦和工程師腦打架，頻繁換方向工作時只開執行板
只估程式時間低估美術／音效／測試耗時每個 feature 五維度都掃一遍
WIP 塞滿什麼都開始、什麼都沒完成In Progress 上限 ≤ 3
花 >30min 選工具偽裝成工作但零產出先用 Trello，不夠再換

執行板標準欄位

欄位意義限制
Backlog已拆好、隨時可開工的任務池—
Ready本週期打算執行的任務—
In Progress正在進行中≤ 3 個
Done已完成，無需再動—
💡 工作時感到迷茫，不是因為不夠努力——而是下一個任務不夠小、不夠清楚。
