# 既有知識與經驗基礎

學習者具備三年前端工程師經驗，有一款安裝數 1000+ 的 Greasyfork userscript 開源專案——證明其具備程式開發能力與獨立交付小產品的經驗。Godot 開發經驗僅限於 toy project 層級，但目前專案的核心 framework/systems 已完成。對專案管理方法論（Agile、Scrum 等）有基本認知但未曾系統性學習或應用於遊戲開發。偏好先學實用技巧再補理論，排斥過重的流程框架。

**Implications**: 不需要從零教程式開發或版本控制。可直接進入遊戲開發特有的管理挑戰（內容生產、關卡設計 pipeline、創意工作的估算）。教學中應避免重複他們已知的一般性軟體工程知識，聚焦在「遊戲開發 vs Web 開發」的差異點上。
