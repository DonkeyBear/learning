# Teaching Notes

## Language
- 使用者以繁體中文溝通，所有課程和參考文件應以中文撰寫
- 專有名詞（如 Backlog、Sprint、Kanban）保留英文原文，輔以中文說明

## Visual
- **深色主題**：所有課程 HTML 和參考文件使用深色背景（--bg: #1a1a1f）
- 避免使用 Tufte-style 浮動側邊欄（sidenote with negative margin），改用 inline callout box（.callout），確保在所有螢幕寬度下排版正常
- 列印時自動切換為淺色主題（@media print）

## Reference Documents（參考文件規範）
- 參考文件是純查閱用途，與課程 HTML 是不同類別的文件
- **禁止**放入互動式測驗、練習題、問答區塊（那是課程的內容）
- 應更緊湊、更偏向「列印友善」：排版密度高、掃讀性強
- 視覺上要與課程有區別——更少的「教學引導」語氣，更多的是「濃縮知識單元」
- 適合的內容：速查表、字彙表、流程圖、Checklist、模板

## Learning Preferences
- **先技巧後理論**：每堂課應從一個可立即使用的實用技巧開始，理論作為補充說明
- **輕量優先**：任何推薦的方法必須能在 5 分鐘內解釋清楚其核心
- **尊重自由度**：不推薦需要嚴格遵守規則的重型框架；強調「調整以適合自己」而非「照做」
- **務實導向**：每堂課必須直接對應到 MISSION.md 中的具體目標
