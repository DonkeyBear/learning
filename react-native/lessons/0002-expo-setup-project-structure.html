🛠️ Lesson 0002 — Expo 開發環境與專案結構

約 30 分鐘 · 難度：基礎＋實作 · 建立於 2026-06-12

為什麼這很重要

上堂課你學到了 RN 的 runtime 架構。現在你要把這個架構「叫出來」——在你的電腦上建立一個真正能跑的 Expo 專案。 這堂課不只教你下指令，而是要讓你理解每個檔案做什麼、為什麼這樣組織。 理解專案結構 = 知道 bug 發生時該看哪個檔案。

📋 前置需求

開始之前，確保你已安裝：

  • Node.js 18+ — 建議用 nvm 或 fnm 管理版本
  • macOS 上建議安裝 watchman（檔案監控工具，Metro bundler 用它追蹤檔案變更）
  • 一台實體手機安裝 Expo Go app，或 macOS 上的 Xcode Simulator / Android Emulator
💡 你需要 Xcode 或 Android Studio 嗎？
初期不需要。安裝 Expo Go app 在實體手機上，掃描 QR code 就能預覽。等到需要自訂 native module 時再安裝 dev client。
但如果你要用 simulator：Xcode（macOS only）或 Android Studio（任何平台）還是要裝。

🚀 第一步：建立專案

打開終端機，執行：

# 建立新專案
npx create-expo-app@latest MyFirstApp

# 進入專案目錄
cd MyFirstApp

# 啟動開發伺服器
npx expo start

npx create-expo-app 做了什麼？

  1. 下載最新的 Expo SDK template
  2. 產生 package.json（依賴清單）
  3. 產生 app.json（Expo 設定檔）
  4. 產生 tsconfig.json（TypeScript 設定）
  5. 產生 App.tsx（進入點元件）
  6. 執行 npm install

📁 專案結構詳解

這是你剛建立的目錄結構。每個檔案都有它的角色：

MyFirstApp/
├── app.json              ← Expo 設定檔（app name、icon、splash...）
├── App.tsx               ← 進入點元件（你的第一個畫面）
├── package.json          ← npm 依賴與 scripts
├── tsconfig.json         ← TypeScript 編譯設定
├── node_modules/         ← 安裝的套件（不要手動修改）
├── assets/               ← 靜態資源（圖片、字型）
├── .gitignore            ← Git 忽略規則
└── ...

核心檔案逐個看

檔案角色你需要改嗎？
app.json Expo 的中央設定檔。定義 app name、slug、version、icon、splash screen、platform-specific 設定 ✅ 每個專案都會改
App.tsx React Native 的根元件。相當於 web React 的 index.tsx + App.tsx 合體 ✅ 你的程式碼起點
package.json 定義 npm scripts（start、ios、android）和依賴版本 ⚠️ 加新套件時會改
tsconfig.json TypeScript 編譯選項。Expo 預設有合理的設定 很少需要改

app.json 重點欄位

{
  "expo": {
    "name": "MyFirstApp",        // app 顯示名稱
    "slug": "MyFirstApp",        // URL-safe 識別碼（EAS 用）
    "version": "1.0.0",          // app 版本
    "orientation": "portrait",   // 螢幕方向
    "icon": "./assets/icon.png", // app 圖示路徑
    "userInterfaceStyle": "light", // 系統主題
    "splash": {                  // 啟動畫面設定
      "image": "./assets/splash.png",
      "resizeMode": "contain",
      "backgroundColor": "#ffffff"
    },
    "ios": { "supportsTablet": true },
    "android": { "adaptiveIcon": { /* ... */ } },
    "plugins": []                // Expo config plugins
  }
}

🏃 第二步：在手機上執行

有三種方式可以預覽你的 app：

方式優點限制
Expo Go (實體手機) 最快、不用裝 Xcode/Android Studio 不能跑自訂 native module
iOS Simulator Mac 內建、開發迭代快 Mac only、不能測相機/推播等硬體功能
Android Emulator 跨平台可用 啟動慢、佔記憶體
📱 最推薦初學者：
手機安裝 Expo Go（App Store / Google Play 搜尋），然後在終端機跑 npx expo start，掃描終端機顯示的 QR code。app 會立即在手機上開啟。

🖐️ 第三步：動手修改 App.tsx

打開 App.tsx，你會看到一個預設的 template。試著改成這樣：

import { StatusBar } from 'expo-status-bar';
import { StyleSheet, Text, View } from 'react-native';

export default function App() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Hello, React Native!</Text>
      <Text style={styles.subtitle}>這是我的第一個 RN App</Text>
      <StatusBar style="auto" />
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#161618',
    alignItems: 'center',
    justifyContent: 'center',
  },
  title: {
    fontSize: 28,
    fontWeight: '700',
    color: '#e4e4e4',
  },
  subtitle: {
    fontSize: 16,
    color: '#8b8b8b',
    marginTop: 8,
  },
});

存檔後，Expo Go 會自動熱重載（Fast Refresh）。這就是 RN 開發的日常 — 改 code → 存檔 → 手機即時看到變化。

🔄 Metro Bundler

當你跑 npx expo start 時，幕後發生的事：

flowchart TD
    Code["你的 code (App.tsx)"]
    Metro["Metro Bundler
JS bundler（類似 webpack / Vite，但是專為 RN 設計）"]
    Hermes["Hermes
(在手機上執行 bundle)"]

    Code --> Metro
    Metro -->|"打包成一個 JS bundle"| Hermes

關鍵差異：

  • Web 用 Vite/webpack → 輸出 HTML+JS+CSS → 瀏覽器
  • RN 用 Metro → 輸出 JS bundle → Hermes（手機上的 JS engine）
  • Metro 不做 tree-shaking、不做 code splitting（這些在 native 世界不適用）
  • Fast Refresh 是 Metro 的功能，不是 Expo 的功能
📝 命名慣例：
Expo SDK 版本號碼跟 RN 版本是對齊的。例如 Expo SDK 52 ≈ RN 0.76。用 npx expo --version 查看你安裝的版本。
查相容性：docs.expo.dev/versions/latest/
