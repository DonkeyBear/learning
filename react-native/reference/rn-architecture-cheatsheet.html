Web vs RN：核心差異

面向React WebReact Native
渲染目標DOM (HTML elements)原生元件 (UIView / ViewGroup)
JS EngineV8 (Chrome), JSC (Safari), etc.Hermes (預設), JSC (iOS fallback)
JS↔UI 通訊同步 DOM API (同 process)JSI C++ 層 (跨 thread/語言)
樣式系統CSS (cascade, inheritance)StyleSheet (subset of CSS, no cascade)
導航URL routingNative stack/tab navigator
佈局引擎Browser layout engineYoga (Flexbox, C++ 實作)

新舊架構對比

元件舊架構 0.75-新架構 0.76+
JS↔Native 通訊 Async Bridge（JSON 序列化、非同步、佇列） JSI（C++ 層、同步、無序列化、雙向）
Native Modules 啟動時全部載入、手寫膠水碼 TurboModules（lazy-load、Codegen 自動產生 binding）
渲染器 UIManager (paper renderer) Fabric（C++ Shadow Tree、Concurrent Mode）
型別安全 執行時錯誤 Codegen（編譯時型別檢查、自動產生 native code）

新架構四大支柱

支柱一句話解釋解決什麼問題
JSI 輕量 C++ 介面，讓 JS 直接呼叫 native 舊 bridge 的 JSON 序列化瓶頸 & 非同步限制
Fabric 新的 C++ 渲染引擎，Shadow Tree + Yoga 舊 UIManager 的跳幀、無法同步讀取 layout
TurboModules lazy-load 的 native module 系統 舊 module 全部在啟動時載入，拖慢 TTI
Codegen 從 TS spec 自動產生 native binding 手寫 native 膠水碼容易出錯、維護成本高

Expo 在這個架構中

Expo 組件角色
Expo SDK預封裝的 native module 集合（無需手寫 native code）
Expo Go開發期預覽 app（不能跑自訂 native module）
Dev Client生產級開發用 app（支援自訂 native module + 熱重載）
expo prebuild從 app.json 產生 ios/ 和 android/ 原生目錄
EAS Build雲端建置（不需本機 Xcode / Android Studio）
EAS UpdateOTA 更新（跳過商店審核推送 JS bundle）

關鍵詞彙

詞彙定義
HermesMeta 為 mobile 設計的輕量 JS engine，支援 AOT compilation，減少 APK 大小和記憶體用量
YogaMeta 開發的跨平台 Flexbox 佈局引擎，C++ 實作，iOS 和 Android 共用同一份程式碼
Shadow Tree在背景 thread 維護的 UI 元件樹，Yoga 在此計算 layout 後才 commit 到 UI thread
JSIJavaScript Interface — 讓 JS 可以直接持有和操作 C++ 物件的輕量 API 層
Codegen從 TypeScript 介面定義自動產生 Java/Kotlin/ObjC/C++ binding code 的工具
Fabric新渲染系統的 codename，包含 Shadow Tree、Yoga、和 native view mounting
