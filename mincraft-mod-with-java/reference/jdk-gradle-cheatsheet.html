📋 JDK & Gradle 速查表

常用指令、目錄結構、術語對照 —— 開發時快速查閱

核心術語

術語說明TS/JS 對照
JDKJava Development Kit：編譯器 + JRE + 開發工具Node.js + tsc + npm
JREJava Runtime Environment：只含執行環境Node.js runtime（不含 npm）
JVMJava Virtual Machine：執行 bytecode 的虛擬機V8 engine
javacJava 編譯器：.java → .classtsc
bytecode編譯後的中間碼（.class 檔案）編譯後的 .js（概念類似）
Gradle建構工具 + 套件管理npm + webpack/vite 的合體
gradlewGradle Wrapper：隨專案攜帶的 Gradlenpx
Maven另一個建構工具（Gradle 的競爭者）—

IDE 選擇

IDE適合對象備註
IntelliJ IDEA Community✅ 所有人（預設推薦）免費開源，Java + Gradle + Git 完全夠用
IntelliJ IDEA Ultimate專業開發者付費，多了 SpringBoot/DB/JS 深度整合
VS Code + Extension Pack不想換編輯器的前端需裝 Java 擴充套件，體驗略遜於 IntelliJ
Eclipse老派開發者 / 遺留專案曾經的霸主，逐漸被 IntelliJ 取代

IntelliJ 常用快捷鍵

快捷鍵功能
Cmd+; / Ctrl+Alt+Shift+S開啟 Project Structure（管理 SDK）
Cmd+N / Alt+Insert產生新檔案 / 自動產生程式碼
Shift Shift萬能搜尋（檔案、類別、設定）
Cmd+Shift+Enter自動補完當前行（加括號、分號）
Cmd+B / Ctrl+B跳到定義（Go to Definition）
Cmd+Shift+A搜尋所有 Action（找不到功能時按這個）

常用 Gradle 指令

指令用途
./gradlew run編譯並執行主程式
./gradlew build編譯 + 跑測試 + 打包成 .jar
./gradlew test只跑測試
./gradlew clean清除建構產物（build/ 目錄）
./gradlew tasks列出所有可用的 task
./gradlew dependencies顯示依賴樹

Gradle 專案目錄結構

my-project/
├── build.gradle.kts       # 建構腳本（依賴、plugin、設定）
├── settings.gradle.kts    # 專案名稱 + 子專案宣告
├── gradlew                # Gradle wrapper script
├── gradle/
│   └── wrapper/
│       └── gradle-wrapper.properties  # Gradle 版本鎖定
└── src/
    ├── main/java/         # 主要原始碼
    ├── main/resources/    # 資源檔（設定檔、圖片等）
    └── test/java/         # 測試原始碼

build.gradle.kts 基本範例

plugins {
    id("java")                              // Java plugin
}

group = "com.example"
version = "1.0.0"

repositories {
    mavenCentral()                          // 套件來源（類似 npm registry）
}

dependencies {
    testImplementation("org.junit.jupiter:junit-jupiter:5.10.0")
    // 格式: "group:artifact:version" （類似 npm 的 package@version）
}

Maven Coordinate

Java 套件的識別格式：
group:artifact:version
例：com.google.guava:guava:33.0.0-jre
相當於 npm 的 package@version

常用 Repository

mavenCentral() — 最大套件庫
maven { url = "..." } — 自訂來源
Fabric MOD 開發會用到專屬的 repository

JDK 版本選擇指南

版本LTS適用場景
Java 21✅推薦：Minecraft 1.21+、SpringBoot 3.x、大多數新專案
Java 17✅Minecraft 1.18–1.20、部分企業環境
Java 8✅老舊專案維護（仍大量存在於企業）
Java 22+❌非 LTS，嘗鮮用。每 6 個月出新版
