# Teaching Pattern Established: "順帶一提" Side-Knowledge Tips

User requested that every lesson include 3–6 "順帶一提" (by-the-way) tip boxes throughout the content. These cover adjacent-but-not-core knowledge — ecosystem comparisons, historical context, best practices, naming conventions, tool comparisons, "why" explanations, etc. The `.tip` CSS class uses a purple left-border accent on a compact dark card.

**Evidence**: User explicitly requested this after reviewing Lesson 0001. Six tips were retroactively added to Lesson 0001 as a demonstration.

**Implications**:
- All future lessons must include `.tip` boxes
- The CSS pattern is standardized and should be reused
- Each tip should be 1–3 sentences max, placed after the related section
- Good candidates: tool comparisons (Gradle vs Maven, Fabric vs NeoForge), historical context, naming conventions, "why" explanations
- See [[NOTES.md]] and persistent memory [[by-the-way-tips-in-lessons]] for full specification
