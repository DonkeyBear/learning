LESSON 0004 / ITEM / ~35 min

第一個物品：註冊你的專屬神劍

理解 Minecraft 核心的 Registry 機制，並將一把自訂的劍加進遊戲世界中。

Minecraft 的大腦：Registry 註冊表

如果你要在一個已經寫好的遊戲中加入新東西（比如一個物品），你不能只寫一個 Class 就算了，因為遊戲引擎根本不知道它的存在。你必須把它登記 (Register) 到遊戲的大腦裡。

Minecraft 中的這個大腦叫做 Registry。遊戲中的所有物品、方塊、生物、音效，全部都在各自的 Registry 裡有一個獨一無二的 ID（例如 minecraft:apple 或 my_mod:magic_sword）。

💡 順帶一提：Identifier 與 ResourceKey

Minecraft 大量使用 Identifier (簡稱 ID) 來尋找資源。格式永遠是 命名空間:資源名稱。從 Minecraft 26 開始，我們必須先透過 Identifier.parse("...") 建立 ID，然後用它生成一個 ResourceKey（註冊金鑰）。這個金鑰就像是一把鎖匙，在建立物品與註冊物品時都必須使用同一把鎖匙，這樣遊戲大腦才不會搞混！

步驟一：撰寫物品與註冊程式碼

建立一個統一管理物品的 Class

在你的主程式 (例如 ExampleMod.java) 旁邊，建立一個新類別叫做 ModItems.java。把所有自訂物品都集中管理是很好的習慣。

package com.example.item; // 替換成你的 package

import net.minecraft.world.item.Item;
import net.minecraft.core.Registry;
import net.minecraft.core.registries.BuiltInRegistries;
import net.minecraft.core.registries.Registries;
import net.minecraft.resources.Identifier;
import net.minecraft.resources.ResourceKey;

public class ModItems {
    // 1. 提早建立專屬的「註冊金鑰 (ResourceKey)」
    public static final ResourceKey<Item> MAGIC_SWORD_KEY = ResourceKey.create(
            Registries.ITEM,
            Identifier.parse("myfirstmod:magic_sword")
    );

    // 2. 實例化你的物品 (必須呼叫 setId 將金鑰綁定進 Properties 中)
    public static final Item MAGIC_SWORD = new Item(
            new Item.Properties().setId(MAGIC_SWORD_KEY)
    );

    // 3. 建立註冊方法
    public static void registerModItems() {
        // 新版的註冊可以直接傳入剛剛做好的 Key
        Registry.register(
                BuiltInRegistries.ITEM,
                MAGIC_SWORD_KEY,
                MAGIC_SWORD
        );
    }
}
💡 順帶一提：public static final 是什麼？

在 Java 中，public static final 是定義常數 (Constants) 的標準語法。public 代表大家都能存取，static 代表它屬於 Class 本身（不需要 new 就能用），final 代表它的參考不可改變（類似 JS 的 const）。在 Minecraft 模組開發中，你幾乎所有的物品實例都會被宣告為 public static final！

在主入口點呼叫註冊方法

打開你的主程式（繼承了 ModInitializer 的那個，通常是 ExampleMod.java），在 onInitialize() 方法中呼叫你剛剛寫的方法：

public class ExampleMod implements ModInitializer {
    public static final String MOD_ID = "myfirstmod";

    @Override
    public void onInitialize() {
        // 將我們的物品註冊進遊戲大腦！
        ModItems.registerModItems();
        System.out.println("Hello Fabric world!");
    }
}
💡 順帶一提：單例模式 (Singleton) 概念

因為 MAGIC_SWORD 是 static final 的實例，這代表遊戲中「這種類型的劍」在記憶體裡永遠只有一份物件拷貝！每個玩家拿在手上的那把劍，其實是記錄在另一個類別 ItemStack (物品堆疊) 裡，而 ItemStack 會指回這唯一的 Item 實例來查詢它的屬性。

步驟二：給它名字與圖案 (Assets)

如果你現在開遊戲，你會能用指令取得劍（/give @s myfirstmod:magic_sword），但它會沒有名字（顯示 item.myfirstmod.magic_sword）且圖案是黑紫相間的除錯材質。我們需要加入資源檔。

📢 重要：資源路徑非常嚴格！
所有的翻譯與材質都必須放在 src/main/resources/assets/[你的modid]/ 下面。任何拼寫錯誤都會導致遊戲找不到檔案。

1. 翻譯檔 (lang)

建立檔案：src/main/resources/assets/myfirstmod/lang/en_us.json

{
  "item.myfirstmod.magic_sword": "Epic Magic Sword"
}
💡 順帶一提：為什麼要用翻譯檔？

Minecraft 內建了強大的 i18n (國際化) 系統。遊戲永遠不會在程式碼裡寫死 "Magic Sword"。程式碼只知道 item.myfirstmod.magic_sword 這個字串鍵值，遊戲會根據玩家當前選擇的語言，去對應的 JSON 檔（例如繁體中文是 zh_tw.json）尋找真正的顯示文字。

2. 物品圖案的三層架構 (Data-Driven Models)

從 Minecraft 26 開始，物品的顯示採用了「資料驅動」的三層架構。要讓劍顯示出來，你需要建立三個檔案，把它們當成「路由 -> 元件 -> CSS」的關係來理解會很快：

A. 物品模型定義檔 (Item Definition)： 類似 Router，負責告訴遊戲這個物品要對應到哪個幾何模型。
建立路徑：src/main/resources/assets/myfirstmod/items/magic_sword.json

{
  "model": {
    "type": "minecraft:model",
    "model": "myfirstmod:item/magic_sword"
  }
}

B. 幾何模型檔 (Model)： 類似 Component，定義物品的 3D 形狀與套用的貼圖變數（注意 parent 必須是 minecraft:item/generated）。
建立路徑：src/main/resources/assets/myfirstmod/models/item/magic_sword.json

{
  "parent": "minecraft:item/generated",
  "textures": {
    "layer0": "myfirstmod:item/magic_sword"
  }
}

C. 實體貼圖 (Texture)： 類似 CSS / 靜態資源檔，真正的 16x16 點陣圖。
準備一張 PNG 圖片，命名為 magic_sword.png，放到：
src/main/resources/assets/myfirstmod/textures/item/magic_sword.png

💡 順帶一提：為什麼要拆成三層？

在前端開發中，我們習慣把路由、邏輯、結構與樣式分離。Minecraft 26 這麼做是為了獲得強大的擴充性！你可以在 Item Definition (A) 撰寫簡單的狀態判斷，例如：「如果這把劍處於防禦狀態，就導向防禦用的 Model B；如果快壞了，就導向破損的 Model C」。這樣一來，顯示邏輯被完全轉移到 JSON 處理，不再需要在 Java 程式碼裡寫死了！

動手做 ✋

  1. 按照上述步驟，建立 ModItems 類別並寫好註冊邏輯。
  2. 建立語言檔，並完成 items/、models/、textures/ 這三層資料夾與檔案的建立。隨便找一張 16x16 的圖片當作材質放進去。
  3. 在 IntelliJ 啟動 Minecraft Client。
  4. 進入遊戲，建立一個單人世界（建議開創造模式與允許作弊）。
  5. 按下 T 開啟聊天框，輸入 /give @s myfirstmod:magic_sword。
  6. 看看你的專屬神劍有沒有出現在手上！

🛠️ 除錯小幫手： 如果拿不到物品，檢查 onInitialize 有沒有真的呼叫 registerModItems()。如果材質紫黑相間，檢查 JSON 裡面的拼字（尤其是 myfirstmod 這個 modid 有沒有跟檔案路徑對上）。

課後檢查清單

  • 我理解 Registry 是 Minecraft 註冊新元素的字典
  • 我了解為什麼物品必須是 public static final
  • 我成功讓遊戲啟動且沒有閃退
  • 我能用 /give 指令拿到自己的物品
  • 物品有正確的名稱和材質圖案
