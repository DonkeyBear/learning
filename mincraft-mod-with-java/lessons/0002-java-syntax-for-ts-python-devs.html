LESSON 0002 / SYNTAX / ~25 min

Java 語法速覽：從 TS/Python 視角看 Java

將你已知的 TypeScript/Python 知識映射到 Java，快速掌握變數、類別與函式的核心語法差異。

為什麼要這樣學？

你已經具備前端（JS/TS）與後端（Python）的經驗，所以不需要從「什麼是變數」、「什麼是迴圈」開始學起。最有效率的學習方式是建立知識映射（Knowledge Mapping）：將 Java 的語法直接對應到你已經熟悉的 TS 和 Python 概念。

Java 是一門強型別（Strongly Typed）且純粹物件導向（Pure OOP）的語言。這表示：

🖥️ 一切都在 Class 裡：沒有像 Python 或 JS 那樣可以游離在類別外的全域函式（Global functions）。
📦 嚴格型別宣告：和 TypeScript 類似，但在 Java 中型別宣告在變數前面，而不是後面（例如 String name 而不是 name: string）。
✅ 編譯期把關：編譯器（javac）非常嚴格，這對大型專案（如 SpringBoot 或 Minecraft MOD）是一大優勢。

1. 變數與型別 (Variables & Types)

在 TypeScript 中，你可能會這樣寫：

let count: number = 10;
const playerName: string = "Steve";
let isActive: boolean = true;

在 Java 中，型別寫在變數前面，且字串必須使用雙引號（單引號是給字元 char 用的）：

int count = 10;
final String playerName = "Steve";  // final 類似於 const，表示不可重新賦值
boolean isActive = true;
💡 順帶一提：Java 10 引入的 var

雖然 Java 是嚴格型別，但在 Java 10 之後加入了 var 關鍵字用於區域變數型別推斷。所以你可以寫 var name = "Steve";，編譯器會自動推斷為 String。注意：它和 JS 的 var（函式作用域）完全不同，它仍然是強型別，一旦推斷為 String 就不能再塞數字進去！

基本資料型別（Primitives）vs 物件型別（Objects）

Java 有兩大類資料型別：

  1. Primitives (基本型別)：int, double, boolean, char, long 等。它們小寫開頭，效能極高，但不能是 null。
  2. Objects (物件型別)：String, Integer, Double 等。大寫開頭，它們是類別的實例，可以是 null。
💡 順帶一提：String 為什麼大寫？

在 Java 中，字串（String）不是基本型別，而是一個真正的物件（Class），因此首字母大寫。你可以對它呼叫方法，例如 "Hello".length()。

2. 類別與方法 (Classes & Methods)

在 Java 中，每一個檔案通常只包含一個 public 類別，且檔名必須和類別名稱完全一致。（例如：Player.java 裡面必須是 public class Player）。

TS/Python vs Java 比較

// Java 的寫法
public class Player {
    // 屬性 (Fields)
    private String name;
    private int health;

    // 建構子 (Constructor)
    public Player(String name) {
        this.name = name;
        this.health = 20; // Minecraft 預設血量
    }

    // 方法 (Method) - 注意回傳型別在方法名稱前面
    public void takeDamage(int amount) {
        this.health -= amount;
        System.out.println(name + " took " + amount + " damage!");
    }
    
    // Getter
    public int getHealth() {
        return this.health;
    }
}

幾點重要差異：

  • 建構子名稱必須和類別名稱完全一樣，且沒有回傳型別（甚至不能寫 void），不同於 TS 的 constructor() 或 Python 的 __init__()。
  • 沒有回傳值的方法要標記為 void。
  • 屬性通常設為 private，然後提供 public get...() 和 set...() 方法來存取。這在 Java 叫作 封裝 (Encapsulation)，是極度普遍的慣例。
💡 順帶一提：為什麼要寫一堆 Getter/Setter？

在 JS/TS 中你可能習慣直接 player.health = 10，但在 Java 中直接把屬性設為 public 被視為一種「破壞封裝」的壞習慣。透過 Getter/Setter（player.getHealth()），可以在未來修改內部實作或加入檢查（例如確保血量不低於 0）而不影響外部呼叫。現代 Java (Records) 或 Lombok 工具可以幫你自動生成這些程式碼，我們後面會學到！

3. 陣列與列表 (Arrays & Lists)

這可能是 TS/Python 開發者最容易踩坑的地方！

在 JS 和 Python 中，陣列（Array / List）的大小是動態的，可以隨時 push 或 append。但在 Java 中，原生陣列（Array）的大小是固定的。

// 原生陣列：大小固定為 3，無法擴充！
String[] items = new String[3];
items[0] = "Sword";

如果你想要一個可以動態增減的列表（就像 JS 的 Array 或 Python 的 List），你要使用的是 ArrayList：

import java.util.ArrayList;
import java.util.List;

// <String> 是泛型 (Generics)，和 TS 非常類似！
List<String> inventory = new ArrayList<>();
inventory.add("Diamond Pickaxe"); // 相當於 JS 的 push()
inventory.add("Apple");

int size = inventory.size(); // 相當於 JS 的 length()
String item = inventory.get(0); // 相當於 JS 的 inventory[0]
💡 順帶一提：List vs ArrayList

你可能會注意到我們寫 List<String> list = new ArrayList<>(); 而不是 ArrayList<String> list = new ArrayList<>();。這是 Java 鼓勵的「針對介面寫程式 (Program to an Interface)」原則。List 是介面，ArrayList 是具體實作。未來如果想換成另一種實作（例如 LinkedList），只需要改等號右邊即可。

動手做 ✋

打開上一堂課建立的 hello-java 專案，修改 Main.java 來練習今天學到的語法：

  1. 在 org.example 套件下建立一個新的類別檔案 Item.java。
  2. 在裡面宣告一個 public class Item，給它兩個 private 屬性：String name 和 int quantity。
  3. 建立一個建構子來初始化這兩個屬性。
  4. 寫兩個 public 方法：getName() 和 getQuantity() 回傳這些屬性的值。
  5. 回到 Main.java 的 main 方法中，建立一個 List<Item>，加入幾個 Item 實例。
  6. 使用 for 迴圈把他們印出來！（提示：Java 的 for-each 寫法是 for (Item item : items) { ... }）

🛠️ IntelliJ 魔法： 當你在寫 List 時，如果出現紅字，按 Alt+Enter (Windows/Linux) 或 Option+Enter (Mac)，IntelliJ 會自動幫你 import java.util.List;！
