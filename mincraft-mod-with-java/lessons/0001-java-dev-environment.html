LESSON 0001 / SETUP / ~20 min

Java 開發環境設定

安裝 IntelliJ IDEA + JDK 21 + Gradle，跑起第一個 Java 程式——為 Minecraft MOD 和 SpringBoot 開發打好地基。

為什麼從這裡開始？

在寫任何一行 Java 程式碼之前，你需要一個能編譯和執行 Java 的環境。這就像寫 TypeScript 之前需要 node 和 tsc，寫 Python 之前需要 python3 直譯器。

Java 的世界有一套自己的工具鏈。這堂課以 IntelliJ IDEA 為預設 IDE（現代 Java 開發的業界標準），帶你搞定四件事：

🖥️ IntelliJ IDEA — Java 開發的業界標準 IDE（類比你寫 TS 時用的 VS Code）
📦 JDK 21（Java Development Kit）— Java 的編譯器 + 執行環境
🔧 Gradle — Java 世界的套件管理 + 建構工具（類似 npm + webpack 的合體）
✅ 驗證 — 確認一切就緒，跑第一個 Hello, Minecraft MODder!

💡 給 TS/Python 開發者的快速對照：
JDK ≈ node + tsc（執行期 + 編譯器）
Gradle ≈ npm + webpack（套件管理 + 建構流程）
JVM ≈ Node.js runtime 或 Python interpreter（執行程式碼的虛擬機）

安裝

安裝 IntelliJ IDEA

下載 IntelliJ IDEA。有兩個版本：Community Edition（免費開源，Minecraft MOD 開發 + 基礎 Java 完全夠用）和 Ultimate（付費，多了 SpringBoot/資料庫/JavaScript 等進階支援）。初學選 Community Edition 就對了。

macOS

# 使用 Homebrew（推薦）
brew install --cask intellij-idea-ce

或從官網下載 .dmg 安裝程式手動安裝。

Windows

從官網下載 .exe 安裝程式，下一步到底即可。

Linux

推薦使用 Snap 安裝，或從官網下載 .tar.gz 解壓縮。

第一次啟動時，IntelliJ 會問你要不要匯入舊設定——全新安裝選 "Do not import settings" 即可。

安裝 JDK 21

我們選擇 JDK 21（LTS 長期支援版）。Minecraft 26 需要 Java 21，SpringBoot 3.x 也支援 Java 21——一個版本通吃兩個目標。

方法一：IntelliJ 內建下載（推薦 ✨）

IntelliJ 可以幫你管理 JDK——不需要手動跑指令：

  1. 開啟 IntelliJ，選擇 File → Project Structure → SDK（或快捷鍵 Cmd+; / Ctrl+Alt+Shift+S）
  2. 點 + → Download JDK
  3. Version 選 21，Vendor 選 Eclipse Temurin（或其他你偏好的發行版）
  4. 點 Download，IntelliJ 自動下載並設定好 JAVA_HOME

這對初學最友善——不會弄亂系統環境變數，每個專案可以獨立指定 JDK 版本。

🖥️ 非 IntelliJ 環境（VS Code / Eclipse / 命令列）

如果你偏好命令列或使用其他編輯器，可以手動安裝 JDK：

方法二：命令列安裝

macOS

# 使用 Homebrew
brew install openjdk@21

# 建立 symlink 讓系統能找到 JDK
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk.jdk \\
  /Library/Java/JavaVirtualMachines/openjdk-21.jdk

# 設定環境變數（加到 ~/.zshrc）
export JAVA_HOME="/opt/homebrew/opt/openjdk@21"
export PATH="$JAVA_HOME/bin:$PATH"

Windows

下載 Eclipse Temurin (Adoptium) JDK 21 的 .msi 安裝程式，執行後確認環境變數 JAVA_HOME 已自動設定。

Linux (Ubuntu/Debian)

sudo apt update
sudo apt install openjdk-21-jdk
驗證 JDK 安裝
java --version
# 預期輸出類似：
# openjdk 21.0.x ...

javac --version
# 預期輸出類似：
# javac 21.0.x

如果兩個指令都顯示版本號，JDK 就裝好了 ✅。在 IntelliJ 中也可以到 File → Project Structure → SDK 查看已安裝的 JDK 列表。

💡 順帶一提：JDK 發行版滿天飛

你會看到很多「JDK」：Oracle JDK、OpenJDK、Eclipse Temurin (Adoptium)、Amazon Corretto、Azul Zulu…它們的核心幾乎完全一樣（都來自 OpenJDK 原始碼），差異在於授權條款、更新支援週期、是否包含額外工具。個人開發選 Eclipse Temurin 最省心——免費、無授權陷阱、各平台都有安裝包。

安裝 Gradle

Gradle 可以用 gradle wrapper 的方式隨專案攜帶（推薦），也可以全域安裝：

# macOS
brew install gradle

# 驗證
gradle --version

不過對接下來的課程，我們會用 Gradle Wrapper（./gradlew），不需要手動安裝 Gradle 也能運作——就像 npx 的概念。IntelliJ 內建 Gradle 支援，打開專案時會自動偵測 build.gradle.kts 並提示匯入。

💡 順帶一提：Gradle vs Maven

Java 世界有兩大建構工具：Maven（2004 年誕生，用 XML 寫設定）和 Gradle（2012 年誕生，用 Groovy/Kotlin DSL 寫設定）。Maven 約定優於配置、生態最成熟；Gradle 更靈活、建構速度更快（增量編譯 + 快取）。為什麼我們用 Gradle？Minecraft MOD 開發（Fabric/NeoForge）和現代 SpringBoot 專案都以 Gradle 為主。求職時兩者都可能遇到——觀念互通，只是設定檔格式不同。

💡 順帶一提：IntelliJ IDEA Community vs Ultimate

Community Edition（免費開源）涵蓋純 Java + Gradle + Maven + Git，Minecraft MOD 開發完全夠用。Ultimate（付費，年費 ~$169）多了 SpringBoot 深度整合、資料庫工具、JavaScript/TypeScript 支援等。好消息：學生或開源專案維護者可以免費申請 Ultimate 授權。初學先用 Community，等學到 SpringBoot 時再評估是否升級。VS Code 用戶需裝 Extension Pack for Java 來獲得類似功能。

第一個 Java 專案

方法一：IntelliJ New Project 精靈（推薦 ✨）

  1. 開啟 IntelliJ，點 New Project
  2. Name 填入 hello-java，Location 選一個你放 code 的目錄
  3. Language 選 Java，Build system 選 Gradle (Kotlin)
  4. JDK 選你剛才裝的 JDK 21
  5. 勾選 "Add sample code"（會自動產生 Main.java 和 build.gradle.kts）
  6. 點 Create，完成！
💻 命令列方式（VS Code / 其他編輯器）

如果你用其他編輯器，可以用 gradle init 建立專案：

# 建立專案目錄
mkdir hello-java && cd hello-java

# 初始化 Gradle 專案
gradle init \\
  --type java-application \\
  --test-framework junit-jupiter \\
  --dsl kotlin \\
  --java-version 21

🤔 為什麼用 Kotlin DSL？ Gradle 建構腳本可以用 Groovy（build.gradle）或 Kotlin（build.gradle.kts）。Kotlin DSL 提供更好的 IDE 自動補全和型別檢查，和寫 TypeScript 的感覺更像。放心，這跟在寫 Java 無關——只是設定檔格式。

💡 順帶一提：三種建構設定格式

Java 專案建構設定經歷了三個階段：Maven 的 pom.xml（冗長的 XML）→ Gradle + Groovy（build.gradle，動態語言）→ Gradle + Kotlin DSL（build.gradle.kts，靜態型別 + IDE 支援最好）。Minecraft 的 Fabric 模版使用 build.gradle（Groovy），之後的課程會教怎麼轉成 Kotlin DSL。現階段知道這三種格式的存在就夠了。

初始化完成後，專案目錄結構長這樣：

hello-java/
├── app/
│   ├── build.gradle.kts    // 套件依賴 + 建構設定
│   └── src/
│       ├── main/java/org/example/
│       │   └── Main.java   // ← 主程式入口
│       └── test/java/org/example/
│           └── MainTest.java
├── gradlew                  // Gradle wrapper (Unix)
├── gradlew.bat              // Gradle wrapper (Windows)
└── settings.gradle.kts      // 專案設定

打開 Main.java（如果用 gradle init 則檔案名叫 App.java），內容長這樣：

package org.example;

public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, Minecraft MODder!");
    }
}
💡 順帶一提：Java 命名慣例

Java 社群有嚴格的命名慣例（比 JS/TS 嚴格得多）：類別名用 PascalCase（Main、MyFirstMod），方法/變數名用 camelCase（main、playerName），常數用 UPPER_SNAKE_CASE（MAX_HEALTH），套件名全小寫倒域名（org.example）。這些慣例在面試時會被注意到，建議一開始就養成。

🔍 快速對照： Java 的 public static void main 是所有程式的入口點，相當於 Python 的 if __name__ == "__main__": 或 Node.js 直接執行 node index.js。每個 Java 程式都從 main 開始執行。

💡 順帶一提：為什麼 main 必須是 static？

Java 中，非 static 的方法需要先 new 一個物件才能呼叫（就像 JS 的 class instance method）。但程式剛啟動時還沒有任何物件存在——所以 JVM 需要一個不依賴物件的入口。static 方法屬於「類別本身」而非「類別的實例」，因此不需要先建立物件就能呼叫。這就是 main 永遠是 static 的原因。

執行你的第一個 Java 程式

IntelliJ：打開 Main.java，點 main 方法左邊的 綠色 ▶ 播放按鈕，選 Run 'Main.main()'。下方的 Run 面板會顯示輸出。

💻 命令列方式
./gradlew run

# 輸出：
# Hello, Minecraft MODder!

這堂課的關鍵觀念

🏗️ JDK vs JRE vs JVM —— 三個名詞你會一直看到：
• JVM（Java Virtual Machine）：執行 Java bytecode 的虛擬機
• JRE（Java Runtime Environment）：JVM + 標準庫 = 只能「跑」Java
• JDK（Java Development Kit）：JRE + 編譯器 (javac) + 開發工具 = 能「寫+跑」Java
開發者裝 JDK，終端使用者裝 JRE（或直接 JDK）。

💡 順帶一提：JVM 上的語言家族

JVM 不只跑 Java。Kotlin（Android 官方語言，Spring 官方支援）、Scala（大數據/函數式程式設計）、Groovy（動態語言、Gradle 早期 DSL）、Clojure（Lisp 方言）——這些語言都編譯成和 Java 一樣的 bytecode 在 JVM 上執行，且可以和 Java 程式碼互相呼叫。選 Java 作為第一門 JVM 語言是對的：生態最大、文件最齊、社群最大。將來學 Kotlin 或 Scala 時會發現很多概念相通。

⚙️ 編譯 vs 直譯 —— Java 是編譯語言，這點和 TS → JS 的過程類似，但目標不是 JS 而是 bytecode（.class 檔案），由 JVM 執行。Python 是直譯語言（雖然也有 .pyc），JS 是 JIT 編譯。Java 的流程：
.java → javac → .class (bytecode) → JVM → 執行

動手做 ✋

  1. 下載並安裝 IntelliJ IDEA Community Edition
  2. 用 IntelliJ 內建的 JDK 下載功能安裝 JDK 21（或手動指令安裝）
  3. 用 IntelliJ 的 New Project 精靈建立一個 Gradle + Java 21 專案（或用 gradle init CLI）
  4. 修改 Main.java，讓它印出你的名字和今天的日期
  5. 點 IntelliJ 的 ▶ Run 按鈕（或 ./gradlew run）執行確認
  6. 試著故意把 System.out.println 打成 system.out.println（小寫 s），觀察 IntelliJ 的紅色底線和編譯錯誤訊息

課後檢查清單

  • IntelliJ IDEA Community Edition 已安裝並能正常啟動
  • java --version 回傳 21.x
  • 能在 IntelliJ 中建立一個 Gradle + Java 21 專案
  • 能點 ▶ Run 按鈕（或 ./gradlew run）成功執行 Java 程式
  • 能說出 JDK / JRE / JVM 三者的區別
  • 理解 .java → .class → JVM 執行 的編譯流程
