LESSON 0003 / FABRIC / ~30 min

進入 Minecraft 的世界：Fabric MOD 環境建置

從官方模板開始，在 IntelliJ 中啟動帶有你的專屬模組的 Minecraft 客戶端。

為什麼用 Fabric？

Minecraft 本身不是開源的，也沒有官方的 Mod API。開發者們必須對遊戲進行反編譯、修改位元組碼（Bytecode Injection）才能加入新功能。為了不讓每個人都重新發明輪子，社群開發了 Mod Loader (模組載入器)。

Fabric 是目前最受歡迎的 Mod Loader 之一。它的特色是輕量、更新快（通常 Minecraft 改版後幾天內就會支援）、API 設計現代化，非常適合搭配你剛學到的現代 Java 開發技巧。

💡 順帶一提：Fabric vs NeoForge

Minecraft 模組界長年有兩大陣營：Forge (歷史最悠久、重量級、複雜) 與 Fabric (輕量、現代)。近期 Forge 團隊分裂出了 NeoForge，目前社群基本上是 Fabric 與 NeoForge 雙強鼎立。作為新手，從架構較為清晰、輕量的 Fabric 入手，學習曲線會平緩許多。學會 Fabric 之後，要轉移到 SpringBoot 後端的觀念也會比較順利！

取得 Fabric Example Mod 模板

我們不會從空白資料夾開始寫 MOD，因為要配置反編譯 Minecraft 程式碼的 Gradle 任務非常複雜。官方提供了一個開箱即用的模板。

下載模板專案

請前往 Fabric Example Mod GitHub 頁面。

點擊綠色的 Code 按鈕，選擇 Download ZIP。或者在命令列執行：

git clone https://github.com/FabricMC/fabric-example-mod.git my-first-mod
cd my-first-mod
# 移除原本的 git 紀錄，這樣你就可以建立自己的 repository
rm -rf .git
匯入 IntelliJ IDEA
  1. 打開 IntelliJ IDEA。
  2. 點選 Open，然後選擇你剛剛解壓縮/clone 的 my-first-mod 資料夾。
  3. ⌛ 最重要的一步：等待！

當專案開啟時，IntelliJ 會自動偵測到 build.gradle，並開始下載 Minecraft 主程式、反編譯遊戲程式碼、下載各種依賴。在右下角的進度條跑完之前（可能需要 3~5 分鐘），請不要做任何操作或修改程式碼，不然 IDE 會報一堆紅色的錯誤。

💡 順帶一提：genSources 任務

如果你發現程式碼裡無法點擊進入 Minecraft 原生類別（例如 Block 或 Item），你需要執行 Gradle 的 genSources 任務。這個步驟會將 Minecraft 混淆過的程式碼轉譯成人類可讀的 Java 程式碼，讓你能夠像看開源專案一樣閱讀 Minecraft 的底層邏輯！在 IntelliJ 右側的 Gradle 面板裡，找到 Tasks > fabric > genSources 並點擊兩下執行。

認識重要檔案

專案讀取完畢後，看看左側的專案結構，以下是這三個你未來會最常碰的檔案：

📄 gradle.properties
這是設定檔。你的 Minecraft 版本（例如 Minecraft 26）和 Fabric API 的版本都在這裡設定。

📄 src/main/resources/fabric.mod.json
這個 JSON 是模組的「身分證」。它告訴 Fabric 你的模組叫什麼名字、作者是誰、以及入口程式碼 (Entrypoint) 在哪裡（這很重要！）。

📄 src/main/java/.../ExampleMod.java
這是模組的主程式。你的所有魔法將從這裡開始執行。

💡 順帶一提：為什麼沒有 main 方法了？

在上一課中，我們說 Java 程式從 public static void main 開始。但在 MOD 開發中，遊戲已經在執行了！你的模組是以「插件 (Plugin)」的形式被 Fabric Loader 載入。所以你不會寫 main 方法，而是實作 Fabric 定義的介面（例如 ModInitializer 的 onInitialize 方法）。這種「框架呼叫你的程式」的模式稱為控制反轉 (Inversion of Control, IoC)，這也是 SpringBoot 的核心思想！

啟動遊戲客戶端！

現在，讓我們來跑跑看這個充滿希望的空白模組：

  1. 在 IntelliJ 右上角的「執行設定 (Run Configuration)」下拉選單中。
  2. 你應該會看到一個叫做 Minecraft Client 的選項（如果沒有，展開 Gradle 面板找 Tasks > fabric > runClient）。
  3. 按下綠色的 ▶ 播放鍵！

如果一切順利，你將會看到熟悉的 Mojang 啟動畫面，並且遊戲主畫面的左下角會顯示 "Fabric (Modded)"。

💡 順帶一提：@Override 標註

當你打開 ExampleMod.java，你會看到方法上面有一個 @Override。這叫作標註 (Annotation)。在 TS 裡我們有 @Decorator，在 Python 有 @decorator。在 Java 中，@Override 是告訴編譯器：「我打算覆寫/實作父類別或介面的這個方法」。如果名字打錯了，編譯器會報錯，是非常實用的防呆機制！

動手做 ✋

  1. 按照教學下載 fabric-example-mod。
  2. 在 IntelliJ 中成功匯入專案（等待 Gradle 同步完成，不再有紅字）。
  3. 打開 src/main/java/com/example/ExampleMod.java。
  4. 在 onInitialize() 方法中，把原本的 LOGGER.info(...) 改成印出 "Hello, Minecraft! I am [你的名字]!"。
  5. 點擊執行 Minecraft Client，在遊戲啟動的同時，觀察 IntelliJ 的 Console 輸出視窗，找到你印出的那行字！

🛠️ 遇到困難？ 如果編譯出現奇怪的 Java 版本錯誤，請去 File > Project Structure 確認 SDK 是不是設為 JDK 21（或 17，視你下載的模板對應的 Minecraft 版本而定）。
