LESSON 0006 / EVENTS / ~40 min

讓物品活起來：事件系統與自訂行為

不再只是靜態的方塊和物品——學習 Fabric 事件系統，讓右鍵能夠施法，讓玩家加入世界時觸發魔法。

為什麼需要事件系統？

到目前為止，你的神劍只是一個存在於物品欄裡的靜態物件，它不能做任何事。真正的 MOD 是靠「監聽遊戲中發生的事」來運作的——玩家右鍵揮劍、踩上方塊、進入世界……這些都是「事件 (Events)」。

如果你熟悉前端開發，這個概念幾乎完全一樣：

🌐 JavaScript（你熟悉的）
button.addEventListener(
  'click',
  (event) => {
    console.log('clicked!');
  }
);
⎈ Fabric（你要學的）
ServerPlayerEvents.AFTER_RESPAWN
  .register(
    (player, ...) -> {
      // 玩家重生後觸發
    }
  );

Fabric 的事件 API 完全採用 觀察者模式 (Observer Pattern)——你向一個「事件總線」登記一個 lambda 回呼函式，當遊戲引擎偵測到對應的遊戲行為，就會呼叫你的函式。

💡 順帶一提：Lambda 在 Java 中的語法

你在 JS 裡寫 (x) => x + 1，Java 則寫 (x) -> x + 1。意義完全相同，只是箭頭從 => 換成了 ->。Fabric 的事件回呼大量使用 lambda，所以你的 JS 直覺在這裡完全用得上！

方法 A：覆寫物品的 use() 方法

最直接的方式：讓你的 MagicSword 繼承自 Item，然後覆寫 use() 方法。當玩家右鍵時，遊戲就會呼叫你的邏輯。

建立自訂 Item 類別

之前 MAGIC_SWORD 直接使用 new Item(...)，現在我們把它升級成一個有自己邏輯的子類別：

package com.example.item;

import net.minecraft.world.item.Item;
import net.minecraft.world.item.ItemStack;
import net.minecraft.world.InteractionResultHolder;
import net.minecraft.world.InteractionHand;
import net.minecraft.world.entity.player.Player;
import net.minecraft.world.level.Level;
import net.minecraft.world.effect.MobEffectInstance;
import net.minecraft.world.effect.MobEffects;

public class MagicSwordItem extends Item {

    public MagicSwordItem(Properties properties) {
        super(properties);
    }

    // 覆寫右鍵行為
    @Override
    public InteractionResultHolder<ItemStack> use(
            Level level, Player player, InteractionHand hand) {

        // 只在伺服器端執行（避免客戶端與伺服器邏輯衝突）
        if (!level.isClientSide()) {
            // 給玩家施加速度增益效果 10 秒
            player.addEffect(
                new MobEffectInstance(
                    MobEffects.MOVEMENT_SPEED,
                    200,  // 200 ticks = 10 秒
                    1     // 等級 II
                )
            );
            // 在聊天室顯示訊息
            player.displayClientMessage(
                net.minecraft.network.chat.Component.literal("✨ Magic Sword activated!"),
                true  // true = 顯示在 Hotbar 上方，不進聊天室
            );
        }

        // 告訴遊戲：「這個右鍵操作成功了！」
        return InteractionResultHolder.success(player.getItemInHand(hand));
    }
}
💡 順帶一提：isClientSide() 是什麼？

在 Minecraft 的多人架構中，遊戲邏輯同時在「客戶端 (Client)」與「伺服器端 (Server)」運行。level.isClientSide() 回傳 true 代表現在正在客戶端。給玩家狀態效果這種「改變遊戲狀態」的事情，必須只在伺服器端執行。你可以把它想像成前端的「只在 server action 裡改資料庫，不在純 UI 元件裡做」的原則。

更新 ModItems 使用新類別

回到 ModItems.java，把 new Item(...) 換成 new MagicSwordItem(...)：

// 在 ModItems.java 中
public static final Item MAGIC_SWORD = new MagicSwordItem(
        new Item.Properties().setId(MAGIC_SWORD_KEY)
);

其他地方（資源檔、Loot Table）完全不需要改，因為 ID 還是一樣的。

方法 B：使用 Fabric 全域事件 API

有些情境不適合寫在 Item 類別裡——例如「玩家加入世界時歡迎他」這件事，沒有辦法塞在任何一個物品的 use() 裡。這時候就要使用 Fabric 的 全域事件 API。

這類事件的監聽器通常直接寫在 ExampleMod.java 的 onInitialize() 中：

import net.fabricmc.fabric.api.networking.v1.ServerPlayConnectionEvents;
import net.minecraft.network.chat.Component;

@Override
public void onInitialize() {
    ModItems.registerModItems();
    ModBlocks.registerModBlocks();

    // 監聽：玩家進入伺服器時觸發
    ServerPlayConnectionEvents.JOIN.register((handler, sender, server) -> {
        var player = handler.player;
        player.displayClientMessage(
            Component.literal("⚔ Welcome, " + player.getName().getString() + "! Magic Mod is active."),
            false
        );
    });
}
💡 順帶一提：var 關鍵字（Java 10+）

Java 10 引入了 var 做型別推斷，讓你不用寫冗長的型別名稱。var player = handler.player; 等同於 ServerPlayer player = handler.player;。這和你在 JS/TS 中熟悉的 let / const 型別推斷完全一樣的感覺，但本質上仍是靜態型別（編譯器已經知道型別是什麼）。

💡 順帶一提：Fabric API 事件 vs. Spring 的 ApplicationEvent

你未來學 SpringBoot 時，會看到 @EventListener 標註與 ApplicationEventPublisher。機制幾乎一模一樣：都是「發布者 (Publisher)」觸發一個事件，「訂閱者 (Subscriber)」接收並處理。Fabric 用的是「介面 + 靜態欄位 + 函式式介面」的組合，Spring 則用了更多標註魔法，但思想是一致的。

步驟三：驗證伺服器 vs. 客戶端邊界

這是 Minecraft 模組開發中最容易踩的坑。以下是一個快速判斷表：

🖥 伺服器端（Server Side）做
改變遊戲狀態（給效果、生成實體、改變方塊）
讀取 / 寫入玩家資料
觸發有副作用的操作

💻 客戶端（Client Side）做
渲染特效、粒子效果
播放音效（也可以透過 server packet 觸發）
顯示 UI / HUD 資訊

在 use() 方法裡，你用 if (!level.isClientSide()) 保護伺服器邏輯。如果沒有這個判斷而直接在兩端都執行，很容易造成效果疊加兩次或客戶端崩潰的 bug。

動手做與檢查清單 ✋

  1. 建立 MagicSwordItem.java 並在 use() 中加入你自己想要的邏輯（例如改成給予飛行效果，或噴出閃電）。
  2. 更新 ModItems.java，讓 MAGIC_SWORD 使用你的新類別。
  3. 在 onInitialize() 中加入玩家加入時的歡迎訊息事件監聽器。
  4. 啟動遊戲，拿著劍右鍵確認效果觸發，並確認訊息出現在正確位置。
  • 我知道 Fabric 事件 API 的運作機制等同於 JS 的 addEventListener。
  • 我理解為什麼要用 !level.isClientSide() 保護伺服器邏輯。
  • 右鍵神劍後，玩家確實獲得了速度增益效果。
  • 玩家加入時，歡迎訊息有出現在聊天室中。

⚠️ 常見錯誤： 如果你在客戶端嘗試呼叫某些伺服器專屬的 API（例如 server.getPlayerList()），IntelliJ 會報紅字。只要把邏輯包進 if (!level.isClientSide()) 裡即可。
